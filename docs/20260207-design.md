# 정보보안기사 문제 풀이 웹 애플리케이션 설계서

**작성일**: 2026-02-07

## 1. 프로젝트 개요

### 1.1 목적
정보보안기사 자격증 시험을 준비하는 사용자를 위한 문제 풀이 및 학습 플랫폼 제공

### 1.2 주요 기능
- 과목별 문제 풀이
- 오답 노트 관리
- 학습 진도 확인
- 모의고사 모드

## 2. 기술 스택

| 구분 | 기술 |
|------|------|
| Frontend | React 19, TypeScript |
| Build Tool | Vite |
| Styling | Tailwind CSS v4 |
| State Management | localStorage (클라이언트 저장) |
| Routing | React Router v7 |
| Deployment | GitHub Pages |

## 3. 정보보안기사 시험 구성

### 3.1 필기시험 (5과목)
1. **시스템 보안**: 운영체제, 클라이언트/서버 보안
2. **네트워크 보안**: 네트워크 개요, 프로토콜, 공격/방어
3. **어플리케이션 보안**: 웹, 데이터베이스, 전자상거래 보안
4. **정보보안 일반**: 암호학, 접근통제, 보안 모델
5. **정보보안 관리 및 법규**: 정보보호 관리체계, 관련 법규

### 3.2 실기시험
- 정보보안 실무 (필답형)

## 4. 애플리케이션 구조

```
src/
├── components/       # 재사용 UI 컴포넌트
├── pages/           # 페이지 컴포넌트
├── hooks/           # 커스텀 훅
├── data/            # 문제 데이터 (JSON)
├── types/           # TypeScript 타입 정의
└── utils/           # 유틸리티 함수
```

## 5. 핵심 데이터 구조

### 5.1 문제 (Question)
```typescript
interface Question {
  id: string;
  subject: Subject;           // 과목
  category: string;           // 세부 카테고리
  type: 'multiple' | 'short'; // 객관식/단답형
  question: string;           // 문제 내용
  options?: string[];         // 보기 (객관식)
  answer: string | number;    // 정답
  explanation?: string;       // 해설
}
```

### 5.2 사용자 학습 기록 (UserProgress)
```typescript
interface UserProgress {
  solvedQuestions: string[];  // 푼 문제 ID 목록
  wrongAnswers: string[];     // 오답 문제 ID 목록
  lastStudyDate: string;      // 마지막 학습일
}
```

## 6. 페이지 구성

| 페이지 | 경로 | 설명 |
|--------|------|------|
| 홈 | `/` | 대시보드, 학습 현황 |
| 과목 선택 | `/subjects` | 과목별 문제 풀이 진입 |
| 문제 풀이 | `/quiz/:subject` | 실제 문제 풀이 |
| 오답 노트 | `/wrong-answers` | 틀린 문제 복습 |
| 모의고사 | `/mock-exam` | 실전 모의고사 |

## 7. 배포 전략

- **develop 브랜치**: 개발 및 테스트
- **main 브랜치**: GitHub Actions를 통한 GitHub Pages 자동 배포

## 8. 데이터 관리 전략

### 8.1 정적 데이터 관리 (문제/이론)

GitHub Pages는 정적 호스팅이므로 **별도 서버 없이** 모든 데이터를 정적 파일로 관리합니다.

#### 디렉토리 구조
```
src/data/
├── questions/
│   ├── system-security.json      # 시스템 보안 문제
│   ├── network-security.json     # 네트워크 보안 문제
│   ├── application-security.json # 어플리케이션 보안 문제
│   ├── security-general.json     # 정보보안 일반 문제
│   └── security-management.json  # 정보보안 관리 및 법규 문제
└── theory/
    ├── system-security.json      # 시스템 보안 이론
    ├── network-security.json     # 네트워크 보안 이론
    └── ...
```

#### 문제 JSON 예시
```json
{
  "subject": "system-security",
  "questions": [
    {
      "id": "sys-001",
      "category": "운영체제 보안",
      "type": "multiple",
      "question": "다음 중 리눅스에서 파일 권한을 변경하는 명령어는?",
      "options": ["chmod", "chown", "chgrp", "chattr"],
      "answer": 0,
      "explanation": "chmod는 파일의 읽기/쓰기/실행 권한을 변경하는 명령어입니다."
    }
  ]
}
```

#### 이론 JSON 예시
```json
{
  "subject": "system-security",
  "topics": [
    {
      "id": "sys-theory-001",
      "title": "리눅스 파일 권한",
      "category": "운영체제 보안",
      "content": "리눅스 파일 권한은 소유자(u), 그룹(g), 기타(o)로 구분되며...",
      "keywords": ["chmod", "권한", "rwx"]
    }
  ]
}
```

### 8.2 사용자 데이터 관리

사용자 학습 기록은 **localStorage**에 저장합니다.

```typescript
// 저장 키
const STORAGE_KEYS = {
  PROGRESS: 'user-progress',      // 학습 진도
  WRONG_ANSWERS: 'wrong-answers', // 오답 목록
  BOOKMARKS: 'bookmarks',         // 북마크
};
```

#### 장점
- 서버 불필요 (GitHub Pages 호환)
- 오프라인 사용 가능
- 개인 데이터 보호 (로컬 저장)

#### 단점
- 브라우저/기기 간 동기화 불가
- 브라우저 데이터 삭제 시 초기화

### 8.3 문제 추가 방법

1. 해당 과목의 JSON 파일 편집
2. 새 문제 객체 추가 (id는 고유해야 함)
3. 커밋 & 푸시 → 자동 배포
