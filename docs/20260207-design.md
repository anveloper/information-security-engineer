# 정보보안기사 문제 풀이 웹 애플리케이션 설계서

**작성일**: 2026-02-07

## 1. 프로젝트 개요

### 1.1 목적
정보보안기사 자격증 시험을 준비하는 사용자를 위한 문제 풀이 및 학습 플랫폼 제공

### 1.2 주요 기능
- 과목별 이론 학습
- 과목별 문제 풀이
- 오답 노트 관리
- 학습 진도 확인
- 모의고사 모드

### 1.3 배포 URL
- **프로덕션**: https://security-engineer.anveloper.dev

## 2. 기술 스택

| 구분 | 기술 |
|------|------|
| Frontend | React 19, TypeScript |
| Build Tool | Vite |
| Styling | Tailwind CSS v4, @tailwindcss/typography |
| Content | MDX (@mdx-js/rollup, remark-gfm) |
| State Management | localStorage (클라이언트 저장) |
| Routing | React Router v7 |
| Deployment | GitHub Pages |

## 3. 정보보안기사 시험 구성

### 3.1 필기시험 (5과목)
1. **시스템 보안**: 운영체제, 클라이언트/서버 보안
2. **네트워크 보안**: 네트워크 개요, 프로토콜, 공격/방어
3. **어플리케이션 보안**: 웹, 데이터베이스, 전자상거래 보안
4. **정보보안 일반**: 암호학, 접근통제, 보안 모델
5. **정보보안 관리 및 법규**: 정보보호 관리체계, 관련 법규

### 3.2 실기시험
- 정보보안 실무 (필답형)

## 5. 핵심 데이터 구조

### 5.1 이론 콘텐츠 (Post)
```typescript
interface Post {
  id: string;
  title: string;
  description: string;
  chapter: string;
  order: number;
  component: React.ComponentType;
}
```

### 5.2 문제 (Question)
```typescript
interface Question {
  id: string;
  subject: Subject;           // 과목
  category: string;           // 세부 카테고리
  type: 'multiple' | 'short'; // 객관식/단답형
  question: string;           // 문제 내용
  options?: string[];         // 보기 (객관식)
  answer: string | number;    // 정답
  explanation?: string;       // 해설
}
```

### 5.3 사용자 학습 기록 (UserProgress)
```typescript
interface UserProgress {
  solvedQuestions: string[];  // 푼 문제 ID 목록
  wrongAnswers: string[];     // 오답 문제 ID 목록
  lastStudyDate: string;      // 마지막 학습일
}
```

## 6. 페이지 구성

| 페이지 | 경로 | 설명 |
|--------|------|------|
| 홈 | `/` | 대시보드, 학습 현황 |
| 과목 선택 | `/subjects` | 과목별 문제 풀이 진입 |
| 과목 상세 | `/subjects/:subject` | 이론 학습 / 문제 풀이 탭 |
| 오답 노트 | `/wrong-answers` | 틀린 문제 복습 |
| 모의고사 | `/mock-exam` | 실전 모의고사 |

## 7. 이론 콘텐츠 시스템

### 7.1 MDX 기반 콘텐츠
- 마크다운 + JSX 조합
- remark-gfm으로 GFM 테이블, 취소선 등 지원
- Tailwind Typography 플러그인으로 스타일링

### 7.2 콘텐츠 구조
```
src/content/theory/{과목}/
├── index.ts          # Post 배열 export
├── 01-*.mdx          # 섹션별 MDX 파일
├── 02-*.mdx
└── ...
```

### 7.3 index.ts 예시
```typescript
import Section1 from './01-section.mdx';

export const posts: Post[] = [
  {
    id: '01-section',
    title: '섹션 제목',
    description: '설명',
    chapter: '1',
    order: 1,
    component: Section1,
  },
];
```

## 8. 배포 전략

- **develop 브랜치**: 개발 및 테스트
- **main 브랜치**: GitHub Actions를 통한 GitHub Pages 자동 배포
- **커스텀 도메인**: security-engineer.anveloper.dev

### 8.1 GitHub Actions 워크플로우
```yaml
# .github/workflows/deploy.yml
on:
  push:
    branches: [main]

jobs:
  deploy:
    - pnpm install
    - pnpm run build
    - Deploy to GitHub Pages
```

## 9. 반응형 디자인

### 9.1 지원 화면 크기
- 모바일: 320px ~
- 태블릿: 640px ~
- 데스크톱: 1024px ~

### 9.2 반응형 요소
- 테이블: 가로 스크롤 지원
- 코드 블록: 가로 스크롤 지원
- 폰트 크기: 모바일에서 축소
- 패딩/마진: 화면 크기에 따라 조정

## 10. 데이터 관리 전략

### 10.1 정적 데이터 관리 (이론/문제)
- MDX 파일: 이론 콘텐츠
- JSON 파일: 문제 데이터 (향후)

### 10.2 사용자 데이터 관리
- localStorage에 저장
- 키: `user-progress`, `wrong-answers`, `bookmarks`

#### 장점
- 서버 불필요 (GitHub Pages 호환)
- 오프라인 사용 가능
- 개인 데이터 보호 (로컬 저장)

#### 단점
- 브라우저/기기 간 동기화 불가
- 브라우저 데이터 삭제 시 초기화
