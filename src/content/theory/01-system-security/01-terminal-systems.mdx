# 단말 시스템

단말 시스템은 사용자가 직접 사용하는 끝단(End-point)의 장치를 의미합니다. 네트워크의 최종 접점으로서 보안의 첫 번째 방어선 역할을 합니다.

---

## PC (Personal Computer)

### 데스크톱/노트북 보안 위협

| 위협 유형 | 설명 |
|----------|------|
| 악성코드 | 바이러스, 웜, 트로이목마, 랜섬웨어, 스파이웨어 |
| 비인가 접근 | 물리적 접근, 원격 접근을 통한 정보 유출 |
| 피싱/스피어피싱 | 사회공학적 기법을 이용한 자격증명 탈취 |
| 취약점 악용 | 패치되지 않은 OS/애플리케이션 취약점 공격 |
| 내부자 위협 | 권한 남용, 정보 유출 |

### PC 보안 대책

#### BIOS/UEFI 보안

**BIOS (Basic Input/Output System)**
- 컴퓨터 부팅 시 하드웨어 초기화 및 OS 로드 담당
- 레거시 시스템에서 사용

**UEFI (Unified Extensible Firmware Interface)**
- BIOS의 후속 표준
- GPT 파티션 지원 (2TB 이상 디스크)
- Secure Boot 기능 제공
- 더 빠른 부팅 속도

| 보안 설정 | 설명 |
|----------|------|
| BIOS/UEFI 패스워드 | Supervisor/User 패스워드 설정 |
| Secure Boot | 서명된 부트로더만 실행 허용 |
| 부팅 순서 제한 | USB, CD/DVD 부팅 비활성화 |
| TPM 활성화 | 하드웨어 기반 보안 기능 활성화 |

#### TPM (Trusted Platform Module)

하드웨어 기반의 보안 모듈로 암호화 키를 안전하게 저장합니다.

**주요 기능**
- 암호화 키 생성 및 저장
- 플랫폼 무결성 측정 (PCR: Platform Configuration Register)
- 원격 증명 (Remote Attestation)
- 봉인/개봉 (Sealing/Unsealing)

**TPM 버전**

| 버전 | 특징 |
|-----|------|
| TPM 1.2 | SHA-1 해시, RSA 암호화 |
| TPM 2.0 | SHA-256, RSA/ECC 지원, 더 유연한 구조 |

#### 디스크 암호화

**FDE (Full Disk Encryption)**

| 솔루션 | 운영체제 | 특징 |
|-------|---------|------|
| BitLocker | Windows | TPM 연동, AES-128/256 |
| FileVault 2 | macOS | XTS-AES-128 암호화 |
| LUKS | Linux | dm-crypt 기반, 다중 키 슬롯 |
| VeraCrypt | 크로스플랫폼 | TrueCrypt 후속, 오픈소스 |

**BitLocker 보호 방식**
- TPM only: TPM에 키 저장
- TPM + PIN: TPM과 사용자 PIN 조합
- TPM + 시작 키: TPM과 USB 키 조합
- 복구 키: 48자리 복구 키

#### 소프트웨어 보안

**안티바이러스 vs EDR**

| 구분 | 안티바이러스 | EDR (Endpoint Detection and Response) |
|-----|------------|---------------------------------------|
| 탐지 방식 | 시그니처 기반 | 행위 기반 + 시그니처 |
| 범위 | 알려진 악성코드 | 알려진/알려지지 않은 위협 |
| 대응 | 자동 격리/삭제 | 탐지, 분석, 대응, 포렌식 |
| 가시성 | 제한적 | 엔드포인트 전체 활동 모니터링 |

**애플리케이션 제어**
- 화이트리스트: 허용된 애플리케이션만 실행
- 블랙리스트: 차단된 애플리케이션 실행 금지
- Windows AppLocker, Software Restriction Policies

**패치 관리**
- WSUS (Windows Server Update Services)
- SCCM (System Center Configuration Manager)
- 자동 업데이트 정책 적용
- 취약점 스캔 및 패치 우선순위 지정

#### Windows 보안 기능

| 기능 | 설명 |
|-----|------|
| UAC (User Account Control) | 관리자 권한 요청 시 사용자 확인 |
| Windows Defender | 기본 제공 안티바이러스/방화벽 |
| Windows Firewall | 호스트 기반 방화벽 |
| Credential Guard | 자격증명 격리 보호 |
| Device Guard | 코드 무결성 정책 적용 |
| Windows Hello | 생체인증 (지문, 얼굴, PIN) |

#### Linux 보안 기능

| 기능 | 설명 |
|-----|------|
| SELinux | 강제적 접근 제어 (MAC) |
| AppArmor | 프로파일 기반 애플리케이션 제어 |
| iptables/nftables | 호스트 기반 방화벽 |
| PAM | 플러그형 인증 모듈 |
| sudo | 권한 상승 제어 |
| chroot | 루트 디렉토리 격리 |

## 모바일 (스마트폰, 태블릿)

### 모바일 보안 위협

| 위협 유형 | 설명 |
|----------|------|
| 악성 앱 | 정상 앱으로 위장한 악성코드 |

### 모바일 기기 관리 솔루션

#### MDM (Mobile Device Management)

기업 모바일 기기를 중앙에서 관리하는 솔루션입니다.

**주요 기능**
- 원격 잠금/초기화 (Remote Lock/Wipe)
- 정책 강제 적용 (패스워드 복잡도, 암호화)
- 앱 배포 및 설치 제한
- 위치 추적
- 탈옥/루팅 탐지
- VPN 설정 배포

#### MAM (Mobile Application Management)

업무용 앱만 별도로 관리하는 솔루션입니다.

**주요 기능**
- 앱 컨테이너화
- 앱 간 데이터 공유 제한 (복사/붙여넣기 차단)
- 앱 수준 암호화
- 앱 원격 삭제
- DLP (Data Loss Prevention) 정책 적용

#### MCM (Mobile Content Management)

모바일 기기의 콘텐츠를 관리하는 솔루션입니다.

- 문서 암호화 및 접근 제어
- 안전한 파일 공유
- 콘텐츠 만료 정책

#### EMM (Enterprise Mobility Management)

MDM + MAM + MCM을 통합한 솔루션입니다.

#### UEM (Unified Endpoint Management)

모바일 + PC + IoT 등 모든 엔드포인트 통합 관리

### BYOD (Bring Your Own Device)

개인 소유 기기를 업무에 사용하는 정책입니다.

| 장점 | 단점 |
|-----|------|
| 비용 절감 | 보안 관리 어려움 |
| 사용자 만족도 향상 | 개인정보 침해 우려 |
| 생산성 향상 | 데이터 유출 위험 |

**BYOD 보안 대책**
- 컨테이너화: 업무/개인 영역 분리
- 네트워크 접근 제어 (NAC)
- VPN 필수 사용
- DLP 정책 적용
- 보안 정책 동의서

## IoT (Internet of Things)

### IoT 개요

사물인터넷(IoT)은 센서, 네트워크, 데이터 처리를 통해 물리적 객체를 인터넷에 연결하는 기술입니다.

**IoT 구성요소**
- 디바이스/센서: 데이터 수집
- 게이트웨이: 프로토콜 변환, 데이터 전처리
- 네트워크: 데이터 전송 (Wi-Fi, Zigbee, LoRa, NB-IoT)
- 클라우드/플랫폼: 데이터 저장 및 분석
- 애플리케이션: 사용자 인터페이스

### IoT 보안 위협

| 위협 | 사례 |
|-----|------|
| 기본 자격 증명 | Mirai 봇넷 (2016) - 기본 패스워드 악용 |
| 암호화 부재 | 평문 통신 도청 |
| 펌웨어 취약점 | 업데이트 메커니즘 부재 |
| 물리적 공격 | JTAG, UART 디버그 포트 악용 |
| 공급망 공격 | 제조 과정에서 악성코드 삽입 |

### OWASP IoT Top 10 (2018)

| 순위 | 취약점 |
|-----|--------|
| 1 | 취약한/하드코딩된/추측 가능한 패스워드 |
| 2 | 안전하지 않은 네트워크 서비스 |
| 3 | 안전하지 않은 에코시스템 인터페이스 |
| 4 | 안전한 업데이트 메커니즘 부재 |
| 5 | 안전하지 않거나 오래된 컴포넌트 사용 |
| 6 | 불충분한 개인정보 보호 |
| 7 | 안전하지 않은 데이터 전송 및 저장 |
| 8 | 기기 관리 부재 |
| 9 | 안전하지 않은 기본 설정 |
| 10 | 물리적 보안 강화 부재 |

### IoT 보안 대책

| 영역 | 대책 |
|-----|------|
| 인증 | 기본 패스워드 변경, 강력한 인증 |

## IIoT (Industrial IoT)

### IIoT 개요

산업용 사물인터넷(IIoT)은 제조, 에너지, 운송 등 산업 분야에 IoT를 적용한 것입니다.

**IT vs OT**

| 구분 | IT (Information Technology) | OT (Operational Technology) |
|-----|----------------------------|----------------------------|
| 목적 | 정보 처리 | 물리적 프로세스 제어 |
| 우선순위 | 기밀성 > 무결성 > 가용성 | 가용성 > 무결성 > 기밀성 |
| 수명 | 3-5년 | 15-20년 |
| 패치 | 정기적 | 매우 어려움 (다운타임) |
| 프로토콜 | TCP/IP | Modbus, DNP3, OPC 등 |

### ICS/SCADA 시스템

**ICS (Industrial Control System)**
- 산업 제어 시스템의 총칭

**SCADA (Supervisory Control and Data Acquisition)**
- 원격지 장비 모니터링 및 제어
- HMI (Human Machine Interface)
- RTU (Remote Terminal Unit)
- PLC (Programmable Logic Controller)

### IIoT 보안 위협

| 위협 | 설명 |
|-----|------|
| 레거시 시스템 | 보안 기능 부재, 패치 불가 |
| IT/OT 융합 | 네트워크 연결로 공격 표면 확대 |
| 가용성 요구 | 보안 조치로 인한 다운타임 불가 |
| 물리적 피해 | 사이버 공격이 물리적 피해로 연결 |

**주요 ICS 공격 사례**

| 사례 | 연도 | 설명 |
|-----|------|------|
| Stuxnet | 2010 | 이란 핵시설 원심분리기 공격 |
| BlackEnergy | 2015 | 우크라이나 전력망 공격 |
| TRITON | 2017 | 안전계장시스템(SIS) 공격 |

### IIoT 보안 대책

**네트워크 분리 (Purdue Model)**

| 레벨 | 영역 | 설명 |
|-----|------|------|
| 5 | Enterprise | 비즈니스 시스템 |
| 4 | Site Business | 사이트 비즈니스 시스템 |
| 3.5 | DMZ | IT/OT 분리 |
| 3 | Site Operations | 제조 운영 시스템 |
| 2 | Area Control | 영역 감시 제어 |
| 1 | Basic Control | 기본 제어 (PLC, RTU) |
| 0 | Process | 물리적 프로세스 |

**보안 대책**

| 영역 | 대책 |
|-----|------|
| 네트워크 | IT/OT 분리, 산업용 방화벽, 단방향 게이트웨이 |
| 접근 제어 | 역할 기반 접근 제어, 다중 인증 |
| 모니터링 | 산업용 IDS, 이상 행위 탐지 |
| 패치 관리 | 가상 패치, 보상 통제 |
| 백업/복구 | 설정 백업, 복구 계획 수립 |
| 물리적 보안 | 접근 통제, CCTV |

## 엔드포인트 보안 솔루션

### EPP vs EDR vs XDR

| 구분 | EPP | EDR | XDR |
|-----|-----|-----|-----|
| 명칭 | Endpoint Protection Platform | Endpoint Detection and Response | Extended Detection and Response |
| 기능 | 예방 중심 | 탐지/대응 중심 | 통합 탐지/대응 |
| 범위 | 엔드포인트 | 엔드포인트 | 엔드포인트 + 네트워크 + 클라우드 |
| 분석 | 시그니처 기반 | 행위 분석, 포렌식 | 상관관계 분석 |

### NAC (Network Access Control)

네트워크 접근 제어 솔루션으로 비인가 기기의 네트워크 접근을 차단합니다.

**주요 기능**
- 기기 인증 및 식별
- 보안 상태 점검 (패치, 안티바이러스 등)
- 네트워크 격리/허용
- 게스트 네트워크 관리

**802.1X 인증**
- 포트 기반 네트워크 접근 제어
- 구성요소: Supplicant, Authenticator, Authentication Server
- EAP (Extensible Authentication Protocol) 사용

### DLP (Data Loss Prevention)

데이터 유출 방지 솔루션입니다.

**적용 위치**
- Endpoint DLP: USB, 프린터, 클립보드 제어
- Network DLP: 이메일, 웹 트래픽 모니터링
- Storage DLP: 파일 서버, DB 콘텐츠 검사
