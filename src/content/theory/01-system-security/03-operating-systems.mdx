# 운영체제

운영체제(OS)는 하드웨어와 소프트웨어 자원을 관리하고, 사용자와 시스템 간의 인터페이스를 제공하는 시스템 소프트웨어입니다.

---

## 운영체제 개요

### 운영체제 기능

| 기능 | 설명 |
|------|------|
| 프로세스 관리 | 프로세스 생성, 스케줄링, 종료 |
| 메모리 관리 | 메모리 할당, 가상 메모리 |
| 파일 시스템 관리 | 파일 생성, 읽기, 쓰기, 삭제 |
| 장치 관리 | 하드웨어 장치 제어 |
| 보안 및 보호 | 접근 제어, 인증, 암호화 |
| 네트워크 관리 | 네트워크 통신 지원 |

### 운영체제 분류

| 분류 | 예시 |
|------|------|
| 데스크톱 OS | Windows, macOS, Linux Desktop |
| 서버 OS | Windows Server, Linux Server, Unix |
| 모바일 OS | Android, iOS |
| 임베디드 OS | Embedded Linux, VxWorks, FreeRTOS |
| 실시간 OS (RTOS) | QNX, VxWorks |

### Windows 보안 구성요소

#### 사용자 계정 및 그룹

**기본 계정**

| 계정 | 설명 | SID |
|------|------|-----|
| Administrator | 기본 관리자 계정 | S-1-5-21-...-500 |
| Guest | 게스트 계정 (비활성화 권장) | S-1-5-21-...-501 |
| SYSTEM | 시스템 서비스 계정 | S-1-5-18 |

**기본 그룹**

| 그룹 | 설명 |
|------|------|
| Administrators | 모든 권한 보유 |

| Guests | 제한된 권한 |
| Backup Operators | 백업/복원 권한 |
| Remote Desktop Users | 원격 데스크톱 접속 권한 |

#### 파일 시스템

**NTFS (New Technology File System)**

| 특징 | 설명 |
|------|------|
| 보안 | ACL 기반 접근 제어 |
| 암호화 | EFS (Encrypting File System) |
| 압축 | 파일/폴더 압축 지원 |
| 저널링 | 트랜잭션 로그로 무결성 보장 |
| 할당량 | 디스크 사용량 제한 |

**NTFS 권한**

| 권한 | 설명 |
|------|------|
| Full Control | 모든 권한 |
| Modify | 읽기, 쓰기, 삭제 |
| Read & Execute | 읽기 및 실행 |
| List Folder Contents | 폴더 내용 보기 |
| Read | 읽기 |
| Write | 쓰기 |

**공유 권한 vs NTFS 권한**
- 네트워크 접근 시: 공유 권한과 NTFS 권한 중 더 제한적인 것 적용
- 로컬 접근 시: NTFS 권한만 적용

#### 레지스트리

Windows 설정 정보를 저장하는 계층적 데이터베이스입니다.

**레지스트리 하이브**

| 하이브 | 설명 |
|--------|------|
| HKEY_LOCAL_MACHINE (HKLM) | 시스템 전체 설정 |
| HKEY_CURRENT_USER (HKCU) | 현재 사용자 설정 |
| HKEY_USERS (HKU) | 모든 사용자 프로필 |
| HKEY_CLASSES_ROOT (HKCR) | 파일 연결, COM 객체 |
| HKEY_CURRENT_CONFIG (HKCC) | 현재 하드웨어 설정 |

**보안 관련 레지스트리 경로**
- `HKLM\SAM` - 보안 계정 관리자 (SAM 데이터베이스)
- `HKLM\SECURITY` - 보안 정책
- `HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run` - 시작 프로그램
- `HKLM\SYSTEM\CurrentControlSet\Services` - 서비스 설정

#### 파일 권한

**권한 표기**

```
-rwxr-xr-x  1 owner group  size date filename
```

| 위치 | 의미 |
|------|------|
| 1 | 파일 유형 (-, d, l, c, b) |
| 2-4 | 소유자 권한 (rwx) |
| 5-7 | 그룹 권한 (rwx) |
| 8-10 | 기타 사용자 권한 (rwx) |

**권한 값**

| 권한 | 문자 | 숫자 |
|------|------|------|
| 읽기 | r | 4 |
| 쓰기 | w | 2 |
| 실행 | x | 1 |

**특수 권한**

| 권한 | 숫자 | 설명 |
|------|------|------|
| SetUID | 4000 | 소유자 권한으로 실행 |
| SetGID | 2000 | 그룹 권한으로 실행 |
| Sticky Bit | 1000 | 소유자만 삭제 가능 (/tmp) |

**권한 관련 명령어**

| 명령어 | 설명 |
|--------|------|
| chmod | 권한 변경 |
| chown | 소유자 변경 |
| chgrp | 그룹 변경 |
| umask | 기본 권한 마스크 |

### Linux 보안 기능

**동작 모드**

| 모드 | 설명 |
|------|------|
| Enforcing | 정책 강제 적용 |
| Permissive | 위반 로깅만 (차단 안 함) |
| Disabled | 비활성화 |

**컨텍스트 구조**
```
user:role:type:level
system_u:object_r:httpd_sys_content_t:s0
```

**주요 명령어**

| 명령어 | 설명 |
|--------|------|
| getenforce | 현재 모드 확인 |
| setenforce | 모드 변경 (임시) |
| sestatus | 상태 확인 |
| semanage | 정책 관리 |
| restorecon | 컨텍스트 복원 |
| chcon | 컨텍스트 변경 |

#### AppArmor

**프로파일 모드**

#### iptables / nftables

Linux 커널 방화벽입니다.

**iptables 테이블**

| 테이블 | 설명 |
|--------|------|
| filter | 패킷 필터링 (기본) |
| nat | 주소 변환 |
| mangle | 패킷 수정 |
| raw | 연결 추적 제외 |

**iptables 체인**

| 체인 | 설명 |
|------|------|
| INPUT | 들어오는 패킷 |
| OUTPUT | 나가는 패킷 |
| FORWARD | 전달 패킷 |
| PREROUTING | 라우팅 전 |
| POSTROUTING | 라우팅 후 |

**iptables 예시**
```bash
# SSH 허용
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# 특정 IP 차단
iptables -A INPUT -s 192.168.1.100 -j DROP

# 모든 입력 차단 (기본 정책)
iptables -P INPUT DROP
```

#### 패스워드 정책

**/etc/login.defs**
```
PASS_MAX_DAYS   90
PASS_MIN_DAYS   1
PASS_MIN_LEN    8
PASS_WARN_AGE   7
```

**/etc/security/pwquality.conf**
```
minlen = 8
dcredit = -1
ucredit = -1
lcredit = -1
ocredit = -1
```

#### SSH 보안 설정

**/etc/ssh/sshd_config**
```
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
MaxAuthTries 3
ClientAliveInterval 300
ClientAliveCountMax 0
AllowUsers user1 user2
Protocol 2
```

#### 불필요한 서비스 비활성화

```bash
systemctl disable telnet.socket
systemctl disable rsh.socket
systemctl disable rlogin.socket
systemctl disable rexec.socket
```

## 용어 해설

| 용어 | 풀이 | 설명 |
|------|------|------|
| NTFS | New Technology File System | Windows의 기본 파일 시스템. ACL 기반 접근 제어 지원 |
| EFS | Encrypting File System | NTFS의 파일/폴더 암호화 기능 |
| ACL | Access Control List | 접근 제어 목록. 객체에 대한 권한 정의 |
| 레지스트리 | Registry | Windows 설정 정보를 저장하는 계층적 데이터베이스 |
| 하이브 | Hive | 레지스트리의 최상위 논리적 그룹 (HKLM, HKCU 등) |
| UAC | User Account Control | 관리자 권한 상승 시 사용자 확인 |
| SID | Security Identifier | Windows의 보안 주체 고유 식별자 |
| GPO | Group Policy Object | Active Directory의 그룹 정책 객체 |
| ext4 | Fourth Extended Filesystem | Linux 표준 파일 시스템 |
| XFS | - | 대용량 파일에 적합한 고성능 Linux 파일 시스템 |
| FHS | Filesystem Hierarchy Standard | Linux 디렉토리 구조 표준 |
| UID | User Identifier | Linux 사용자 고유 식별 번호 |
| GID | Group Identifier | Linux 그룹 고유 식별 번호 |
| SetUID | Set User ID | 실행 시 파일 소유자 권한으로 실행 |
| SetGID | Set Group ID | 실행 시 파일 그룹 권한으로 실행 |
| Sticky Bit | 스티키 비트 | 디렉토리 내 파일을 소유자만 삭제 가능하게 설정 |
| SELinux | Security-Enhanced Linux | NSA 개발 강제적 접근 제어(MAC) 시스템 |
| AppArmor | Application Armor | 경로 기반 프로파일 접근 제어 (Ubuntu 기본) |
| iptables | - | Linux 커널 방화벽 (Netfilter) |
| nftables | - | iptables를 대체하는 차세대 Linux 방화벽 |
| PAM | Pluggable Authentication Modules | 모듈식 인증 프레임워크 |
| syslog | System Logging Protocol | 시스템 로그 수집 표준 프로토콜 |
| journald | - | systemd의 로그 수집 데몬 |
| Secure Enclave | 보안 영역 | iOS/macOS의 하드웨어 기반 키 저장소 |
| TEE | Trusted Execution Environment | 신뢰 실행 환경. 격리된 보안 영역 |
| RTOS | Real-Time Operating System | 실시간 운영체제. 시간 제약 처리 |
| Docker | - | 컨테이너 기반 애플리케이션 배포 플랫폼 |
| Kubernetes | K8s | 컨테이너 오케스트레이션 플랫폼 |
| Hypervisor | 하이퍼바이저 | 가상 머신을 생성하고 관리하는 소프트웨어 |
