# 정보보호 관리

## 1. 정보보호 개요

### 1.1 정보보호의 정의

정보의 수집, 가공, 저장, 검색, 송수신 과정에서 정보의 훼손, 변조, 유출 등을 방지하기 위한 관리적, 기술적 수단을 마련하는 것이다.

### 1.2 정보보호의 목표 (CIA Triad)

```
            기밀성
         (Confidentiality)
              ╱╲
             ╱  ╲
            ╱    ╲
           ╱      ╲
    무결성 ──────── 가용성
  (Integrity)   (Availability)
```

| 목표 | 설명 | 위협 | 대책 |
|------|------|------|------|
| 기밀성 | 인가된 자만 접근 | 도청, 유출 | 암호화, 접근통제 |
| 무결성 | 비인가 수정 방지 | 변조, 위조 | 해시, 전자서명 |
| 가용성 | 필요 시 접근 가능 | DoS, 장애 | 이중화, 백업 |

### 1.3 추가적 보안 목표

| 목표 | 설명 |
|------|------|
| 인증 (Authentication) | 신원 확인 |
| 부인방지 (Non-repudiation) | 행위 부인 방지 |
| 책임추적성 (Accountability) | 행위 추적 가능 |

## 2. 정보보호 거버넌스

### 2.1 개념

조직의 정보보호 활동이 비즈니스 목표에 부합하도록 방향을 제시하고 통제하는 체계이다.

### 2.2 정보보호 거버넌스 프레임워크

```
┌─────────────────────────────────────────────────┐
│               이사회/경영진                       │
│          (정보보호 방향 설정, 감독)               │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│              정보보호 위원회                      │
│         (정책 결정, 위험 관리 감독)              │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│            CISO/정보보호 책임자                  │
│          (보안 전략 수립, 실행)                  │
└──────────────────────┬──────────────────────────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
     ┌────────┐   ┌────────┐   ┌────────┐
     │보안 운영│   │보안 기획│   │보안 감사│
     └────────┘   └────────┘   └────────┘
```

### 2.3 정보보호 조직

| 역할 | 책임 |
|------|------|
| 경영진 | 정보보호 의지 표명, 자원 할당 |
| CISO | 정보보호 전략/정책 수립 총괄 |
| 정보보호 관리자 | 보안 통제 운영 |
| 정보보호 담당자 | 실무 수행 |
| 전 직원 | 정책 준수, 보안 인식 |

## 3. 정보보호 정책

### 3.1 정책 체계

```
┌─────────────────────────────────────────┐
│           정보보호 정책 (Policy)          │
│       조직의 보안 방향과 원칙 선언         │
└─────────────────────┬───────────────────┘
                      │
┌─────────────────────▼───────────────────┐
│            표준 (Standards)              │
│         필수 준수 사항 명시               │
└─────────────────────┬───────────────────┘
                      │
┌─────────────────────▼───────────────────┐
│            지침 (Guidelines)             │
│         권고 사항 (준수 권장)             │
└─────────────────────┬───────────────────┘
                      │
┌─────────────────────▼───────────────────┐
│           절차 (Procedures)              │
│        구체적 수행 방법/단계              │
└─────────────────────────────────────────┘
```

### 3.2 정책 구성 요소

| 구성 요소 | 내용 |
|-----------|------|
| 목적 | 정책의 목표 |
| 범위 | 적용 대상 |
| 역할과 책임 | 담당자별 책임 |
| 정책 내용 | 구체적 요구사항 |
| 처벌 조항 | 위반 시 제재 |
| 시행일자 | 효력 발생일 |
| 검토/개정 | 정기 검토 주기 |

### 3.3 주요 정보보호 정책

| 정책 | 내용 |
|------|------|
| 접근통제 정책 | 시스템/데이터 접근 권한 관리 |
| 비밀번호 정책 | 비밀번호 생성/변경 규칙 |
| 이메일 사용 정책 | 업무 이메일 사용 규칙 |

| 모바일 기기 정책 | BYOD, 모바일 보안 |
| 원격 접속 정책 | VPN, 재택근무 보안 |
| 암호화 정책 | 데이터 암호화 요건 |
| 백업 정책 | 백업 주기, 보관 |
| 사고 대응 정책 | 침해사고 대응 절차 |

## 5. 자산 관리

### 5.1 정보자산 분류

| 유형 | 예시 |
|------|------|
| 정보 자산 | 문서, 데이터베이스, 개인정보 |
| 소프트웨어 자산 | 운영체제, 애플리케이션 |
| 물리적 자산 | 서버, 네트워크 장비, PC |
| 서비스 자산 | 네트워크 서비스, 클라우드 |
| 인적 자산 | 직원, 외부 인력 |

### 5.2 자산 분류 등급

| 등급 | 설명 | 예시 |
|------|------|------|
| 1급 (극비) | 유출 시 치명적 피해 | 핵심 기술, 개인정보 |
| 2급 (비밀) | 유출 시 심각한 피해 | 내부 전략 문서 |
| 3급 (대외비) | 유출 시 업무 지장 | 일반 내부 문서 |
| 일반 | 공개 가능 | 홍보 자료 |

### 5.3 자산 관리 수명주기

```
┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
│  획득   │───→│  사용   │───→│  유지   │───→│  폐기   │
└────────┘    └────────┘    └────────┘    └────────┘
    │              │              │              │
    ↓              ↓              ↓              ↓
  등록/분류     접근통제       패치/백업      안전 삭제
```

## 9. 업무 연속성 관리 (BCM)

### 9.1 BCM 개요

Business Continuity Management: 비즈니스 중단 시에도 핵심 업무를 지속할 수 있도록 하는 관리 체계이다.

### 9.2 BCM 프레임워크

```
┌─────────────────────────────────────────────────┐
│                    BCM 정책                      │
└──────────────────────┬──────────────────────────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
    ┌──────────┐ ┌──────────┐ ┌──────────┐
    │   BIA    │ │ 위험 평가 │ │ 전략 수립 │
    └────┬─────┘ └────┬─────┘ └────┬─────┘
         │            │            │
         └────────────┴────────────┘
                      │
          ┌───────────┴───────────┐
          ▼                       ▼
    ┌──────────┐           ┌──────────┐
    │   BCP    │           │   DRP    │
    │(업무연속성)│           │(재해복구) │
    └────┬─────┘           └────┬─────┘
         │                      │
         └──────────┬───────────┘
                    ▼
              ┌──────────┐
              │  테스트   │
              │  /훈련   │
              └──────────┘
```

### 9.3 BIA (Business Impact Analysis)

**목적:** 업무 중단이 조직에 미치는 영향 분석

**주요 지표:**

| 지표 | 설명 |
|------|------|
| MTD | Maximum Tolerable Downtime (최대허용중단시간) |
| RTO | Recovery Time Objective (복구목표시간) |
| RPO | Recovery Point Objective (복구목표시점) |
| WRT | Work Recovery Time (업무복구시간) |

```
        장애 발생                    MTD
            │                        │
   ─────────┼────────────────────────┼─────────→ 시간
            │                        │
   데이터 ←─│─→ 시스템 ←────────────→│
   손실    RPO    복구      RTO     WRT
                  시작              업무
                                   정상화
```

### 9.4 BCP (Business Continuity Plan)

| 구성 요소 | 내용 |
|-----------|------|
| 비상 연락망 | 핵심 인력 연락처 |
| 대체 사이트 | 백업 운영 장소 |
| 핵심 업무 절차 | 우선순위별 복구 절차 |
| 자원 요구사항 | 필요 인력, 장비, 시스템 |
| 커뮤니케이션 | 내/외부 소통 계획 |

### 9.5 DRP (Disaster Recovery Plan)

**복구 사이트 유형:**

| 유형 | 설명 | RTO | 비용 |
|------|------|-----|------|
| Hot Site | 즉시 운영 가능 | 분~시간 | 최고 |
| Warm Site | 일부 장비 보유 | 시간~일 | 중간 |
| Cold Site | 빈 공간만 확보 | 일~주 | 낮음 |
| Mobile Site | 이동형 시설 | 시간~일 | 중간 |
| Cloud Site | 클라우드 기반 | 분~시간 | 가변 |

### 9.6 BCM 테스트

| 테스트 유형 | 설명 |
|-------------|------|
| 체크리스트 | 문서 검토 |
| 워크스루 | 담당자 절차 검토 |
| 시뮬레이션 | 가상 시나리오 훈련 |
| 병행 테스트 | 대체 사이트 운영 |
| 전체 중단 테스트 | 실제 전환 |

## 시험 대비 핵심 포인트

### 정보보호 목표

| 목표 | 영문 | 위협 | 대책 |
|------|------|------|------|
| 기밀성 | Confidentiality | 도청 | 암호화 |
| 무결성 | Integrity | 변조 | 해시 |
| 가용성 | Availability | DoS | 이중화 |

### 정책 체계

```
정책 (Policy) - 방향/원칙
   ↓
표준 (Standards) - 필수 사항
   ↓
지침 (Guidelines) - 권고 사항
   ↓
절차 (Procedures) - 수행 방법
```

### BCM 지표

| 지표 | 의미 | 관계 |
|------|------|------|
| MTD | 최대 허용 중단 시간 | MTD ≥ RTO + WRT |
| RTO | 시스템 복구 목표 시간 | - |
| RPO | 데이터 복구 시점 | 과거로 |
| WRT | 업무 정상화 시간 | - |

### 복구 사이트 비교

| 사이트 | 준비 수준 | 비용 | RTO |
|--------|-----------|------|-----|
| Hot | 즉시 운영 | 高 | 短 |
| Warm | 일부 준비 | 中 | 中 |
| Cold | 장소만 | 低 | 長 |

### 백업 유형

| 유형 | 백업 대상 | 복구 속도 |
|------|-----------|-----------|
| 전체 | 모든 데이터 | 빠름 |
| 증분 | 이전 백업 이후 변경분 | 느림 |
| 차등 | 마지막 전체 이후 변경분 | 중간 |
