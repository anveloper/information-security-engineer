# 정보보안 법규

## 1. 정보보안 법률 체계 개요

### 1.1 국내 정보보안 관련 법률

```
┌─────────────────────────────────────────────────────────────┐
│                        헌법                                  │
│                  (개인정보 자기결정권)                        │
└──────────────────────────┬──────────────────────────────────┘
                           │
    ┌──────────────────────┼──────────────────────┐
    ↓                      ↓                      ↓
┌──────────┐        ┌──────────┐         ┌──────────┐
│개인정보법│        │정보통신망법│         │전자정부법│
└──────────┘        └──────────┘         └──────────┘
    │                      │                      │
    ↓                      ↓                      ↓
┌──────────┐        ┌──────────┐         ┌──────────┐
│신용정보법│        │전자서명법│          │전자문서법│
└──────────┘        └──────────┘         └──────────┘
```

### 1.2 주요 법률 목적

| 법률 | 목적 |
|------|------|
| 개인정보 보호법 | 개인정보 처리 및 보호 |
| 정보통신망법 | 정보통신망 이용 촉진 및 보호 |
| 전자서명법 | 전자서명 법적 효력 |
| 정보통신기반보호법 | 주요정보통신기반시설 보호 |
| 전자정부법 | 전자정부 구현 및 운영 |

## 2. 개인정보 보호법

### 2.1 개요

**목적:** 개인정보의 처리 및 보호에 관한 사항을 정하여 개인의 자유와 권리를 보호

**적용 대상:** 모든 개인정보처리자 (공공/민간)

### 2.2 개인정보 정의

| 유형 | 정의 | 예시 |
|------|------|------|
| 개인정보 | 살아있는 개인을 식별할 수 있는 정보 | 이름, 주민번호, 영상 |
| 민감정보 | 사상, 건강 등 민감한 정보 | 건강정보, 유전정보, 범죄경력 |
| 고유식별정보 | 개인을 고유하게 식별하는 정보 | 주민번호, 여권번호, 운전면허번호 |
| 가명정보 | 추가정보 없이 특정 개인을 알아볼 수 없게 처리한 정보 | - |

### 2.3 개인정보 보호 원칙 (제3조)

| 원칙 | 내용 |
|------|------|
| 수집 제한 | 목적에 필요한 최소한의 정보 수집 |
| 목적 명확화 | 수집 목적 명확히 특정 |
| 목적 내 처리 | 수집 목적 범위 내 처리 |
| 정확성 확보 | 정확하고 최신 상태 유지 |
| 안전성 확보 | 안전하게 관리 |
| 처리 투명성 | 처리 사실 공개 |
| 권리 보장 | 정보주체 권리 보장 |
| 책임 준수 | 개인정보처리자 책임 |

### 2.4 정보주체의 권리

| 권리 | 내용 |
|------|------|
| 열람권 | 자신의 개인정보 열람 요구 |
| 정정권 | 오류 시 정정 요구 |
| 삭제권 | 개인정보 삭제 요구 |
| 처리정지권 | 처리 정지 요구 |
| 동의 철회권 | 동의 철회 |
| 전송요구권 | 다른 개인정보처리자에게 전송 요구 |

### 2.5 개인정보 처리 단계별 조치

#### 수집 단계

| 항목 | 내용 |
|------|------|
| 동의 획득 | 수집 전 동의 (고유식별정보, 민감정보는 별도 동의) |
| 고지 사항 | 수집 목적, 항목, 보유기간, 거부권 등 |
| 최소 수집 | 필요 최소한의 정보만 수집 |
| 법적 근거 | 동의 외 법률 근거 (계약, 법령, 생명/안전) |

#### 이용/제공 단계

| 항목 | 내용 |
|------|------|
| 목적 내 이용 | 수집 목적 범위 내 이용 |
| 제3자 제공 | 별도 동의 또는 법적 근거 필요 |
| 위탁 | 위탁 계약서 작성, 관리 감독 |
| 국외 이전 | 정보주체 동의 또는 보호조치 |

#### 보관/파기 단계

| 항목 | 내용 |
|------|------|
| 보유 기간 | 수집 목적 달성 시까지 (법령 예외) |
| 접근 통제 | 개인정보 접근 권한 관리 |
| 암호화 | 비밀번호, 고유식별정보 암호화 |
| 파기 | 복구 불가능한 방법으로 파기 |

### 2.6 개인정보 보호책임자 (CPO)

**지정 의무:**
- 공공기관: 필수
- 민간: 5만 명 이상 정보주체 처리 시

**업무:**
- 개인정보 보호 계획 수립/시행
- 개인정보 처리 실태 점검
- 개인정보 침해 대응
- 교육/훈련

### 2.7 개인정보 영향평가

**대상 시스템:**
- 50만 명 이상 민감정보/고유식별정보 처리
- 100만 명 이상 개인정보 처리
- 공공기관 개인정보 연계 시스템

**평가 항목:**
- 침해 위험 요인 분석
- 위험성 평가
- 개선 사항 도출

### 2.8 과징금 및 벌칙

| 위반 사항 | 제재 |
|-----------|------|
| 개인정보 유출 | 매출액 3% 이하 과징금 |
| 동의 없는 제3자 제공 | 5년 이하 징역 또는 5천만원 이하 벌금 |
| 안전조치 미흡 | 2년 이하 징역 또는 2천만원 이하 벌금 |
| 목적 외 이용 | 5년 이하 징역 또는 5천만원 이하 벌금 |

## 3. 정보통신망 이용촉진 및 정보보호 등에 관한 법률

### 3.1 개요

**목적:** 정보통신망의 이용 촉진 및 정보통신서비스 이용자 보호

**적용 대상:** 정보통신서비스 제공자, 이용자

### 3.2 정보통신서비스 제공자 의무

| 의무 | 내용 |
|------|------|
| 개인정보 보호조치 | 기술적/관리적 보호조치 이행 |
| 침해사고 대응 | 침해사고 신고 및 대응 |
| 약관 게시 | 이용약관 공개 |
| 청소년 보호 | 청소년유해정보 접근 제한 |

### 3.3 개인정보 보호조치

#### 기술적 보호조치

| 조치 | 내용 |
|------|------|
| 접근 통제 | 개인정보처리시스템 접근 권한 관리 |
| 암호화 | 비밀번호, 주민번호 등 암호화 |
| 접속 기록 | 접속 기록 보관 (1년 이상) |
| 보안 프로그램 | 백신, 방화벽 설치/운영 |

#### 관리적 보호조치

| 조치 | 내용 |
|------|------|
| 내부 관리계획 | 개인정보 보호 계획 수립/시행 |
| 교육 | 정기 보안 교육 |
| 위탁 관리 | 수탁자 관리 감독 |

### 3.4 침해사고 신고

**신고 대상:**
- 정보통신망 침해사고 (해킹, DDoS 등)
- 개인정보 유출 사고

**신고 기한:**
- 침해사고 인지 후 24시간 이내 신고
- 개인정보 유출 시 72시간 이내 통지

**신고 기관:**
- 한국인터넷진흥원 (KISA)
- 개인정보보호위원회 (개인정보 유출 시)

### 3.5 불법 정보 유통 금지

| 금지 정보 | 내용 |
|-----------|------|
| 음란물 | 음란한 부호, 문언, 영상 등 |
| 명예훼손 | 사람을 비방할 목적의 정보 |
| 사이버 스토킹 | 공포심/불안감 유발 정보 |
| 청소년 유해정보 | 청소년에게 유해한 정보 |

## 4. 정보통신기반 보호법

### 4.1 개요

**목적:** 주요정보통신기반시설의 보호

**대상:** 국가 주요 정보통신기반시설

### 4.2 주요정보통신기반시설

**지정 기준:**
- 전자적 침해행위로 국가안전보장에 중대한 영향
- 행정, 국방, 치안, 금융, 통신, 운송, 에너지 분야

**주요 기반시설 예시:**

| 분야 | 예시 |
|------|------|
| 금융 | 은행 전산시스템 |
| 통신 | 통신망, 인터넷 교환노드 |
| 에너지 | 전력, 가스 제어시스템 |
| 교통 | 항공, 철도 운영시스템 |
| 의료 | 의료정보시스템 |
| 수자원 | 상수도 제어시스템 |

### 4.3 관리기관 의무

| 의무 | 내용 |
|------|------|
| 취약점 분석/평가 | 정기 취약점 점검 |
| 보호대책 수립/이행 | 보호계획 수립 및 시행 |
| 침해사고 대응 | 침해사고 대응체계 구축 |
| 보호조치 이행 | 기술적/관리적 보호조치 |

### 4.4 취약점 분석/평가

**주기:** 매년 (또는 중요 변경 시)

**평가 항목:**
- 관리적 취약점
- 기술적 취약점
- 물리적 취약점

## 5. 전자서명법

### 5.1 개요

**목적:** 전자서명의 법적 효력 및 인증체계 확립

### 5.2 주요 정의

| 용어 | 정의 |
|------|------|
| 전자서명 | 전자문서에 서명자를 확인하고 내용에 대한 승인을 나타내는 전자적 정보 |
| 전자서명생성정보 | 전자서명 생성에 이용되는 개인키 |
| 전자서명인증 | 전자서명생성정보가 특정인에게 유일하게 속한다는 사실 확인 |
| 인증서 | 전자서명생성정보가 특정인에게 속한다는 증명서 |

### 5.3 전자서명 효력

**서명날인 효력:**
- 전자서명은 서명 또는 날인과 동일한 효력
- 공인전자서명 폐지 (2020년) → 다양한 전자서명 인정

**효력 불인정 사유:**
- 당사자 간 약정으로 배제
- 법령에서 특정 서명/날인 요구

### 5.4 전자서명인증사업

| 사업자 유형 | 역할 |
|-------------|------|
| 인증기관 | 인증서 발급/관리 |
| 등록기관 | 이용자 등록/신원확인 |
| 검증기관 | 전자서명 유효성 검증 |

**적용 대상:**
- EU 거주자 개인정보 처리 기업
- EU 내 사업장 보유 기업

**주요 원칙:**

### 개인정보 처리 동의 사항

수집 시 고지 필수 항목:
1. 수집 **목적**
2. 수집 **항목**
3. **보유 기간**
4. 동의 거부 **권리** 및 불이익

### GDPR vs 개인정보 보호법

| 구분 | GDPR | 개인정보 보호법 |
|------|------|-----------------|

| DPO | 필수 (일부) | 필수 (대형) |

### 개인정보 영향평가 대상

- 50만 명 이상 **민감정보/고유식별정보**
- 100만 명 이상 **개인정보**
- 공공기관 **연계 시스템**
