# 단말 시스템

단말 시스템은 사용자가 직접 사용하는 끝단(End-point)의 장치를 의미합니다. 네트워크의 최종 접점으로서 보안의 첫 번째 방어선 역할을 합니다.

---

## PC (Personal Computer)

### 데스크톱/노트북 보안 위협

- 악성코드 감염 (바이러스, 웜, 트로이목마, 랜섬웨어)
- 비인가 접근 및 정보 유출
- 피싱/스피어피싱 공격
- 취약한 소프트웨어 악용

### PC 보안 대책

#### BIOS/UEFI 보안

- BIOS 패스워드 설정
- Secure Boot 활성화
- 부팅 순서 제한 (USB, CD 부팅 비활성화)

#### 디스크 보안

- 전체 디스크 암호화 (BitLocker, FileVault)
- SSD Secure Erase
- TPM(Trusted Platform Module) 활용

#### 소프트웨어 보안

- 안티바이러스/EDR 설치 및 실시간 감시
- 운영체제 및 애플리케이션 최신 패치 유지
- 불필요한 소프트웨어 제거
- 애플리케이션 화이트리스트 적용

#### 네트워크 보안

- 개인 방화벽 활성화
- 공용 Wi-Fi 사용 시 VPN 필수
- 불필요한 네트워크 서비스 비활성화

---

## 모바일 (스마트폰, 태블릿)

### 모바일 보안 위협

- 악성 앱 설치
- 루팅/탈옥으로 인한 보안 약화
- 분실/도난 시 정보 유출
- 안전하지 않은 Wi-Fi 연결
- SMS 피싱 (스미싱)

### 모바일 보안 대책

#### MDM (Mobile Device Management)

기업 모바일 기기를 중앙에서 관리하는 솔루션입니다.

- 원격 잠금/초기화 기능
- 정책 강제 적용 (패스워드, 암호화)
- 앱 설치 제한
- 위치 추적

#### MAM (Mobile Application Management)

- 업무용 앱만 별도 관리
- 앱 간 데이터 공유 제한
- 앱 수준 암호화

#### 기본 보안 설정

- 화면 잠금 설정 (PIN, 패턴, 생체인증)
- 기기 암호화 활성화
- 원격 추적/삭제 기능 활성화
- 공식 앱스토어만 사용
- 루팅/탈옥 금지

---

## 프린터/복합기

### 프린터 보안 위협

- 네트워크를 통한 무단 접근
- 인쇄 데이터 도청
- 저장된 문서 유출
- 펌웨어 취약점 악용

### 프린터 보안 대책

- 관리자 기본 패스워드 변경
- 불필요한 프로토콜 비활성화 (Telnet, FTP)
- HTTPS 관리 인터페이스 사용
- 인쇄 작업 암호화
- 저장된 문서 자동 삭제 설정
- 펌웨어 최신 버전 유지
- 인쇄 로그 감사

---

## IoT (Internet of Things)

### IoT 보안 위협

- 기본 자격 증명 사용
- 암호화되지 않은 통신
- 펌웨어 업데이트 부재
- 물리적 접근을 통한 공격
- 봇넷 감염 (Mirai 등)

### IoT 보안 대책

- 기본 패스워드 즉시 변경
- 네트워크 분리 (IoT 전용 VLAN)
- 통신 암호화 (TLS)
- 정기적 펌웨어 업데이트
- 불필요한 기능/포트 비활성화
- 모니터링 및 이상 행위 탐지

---

## IIoT (Industrial IoT)

### IIoT 특수 고려사항

- 운영 기술(OT)과 정보 기술(IT) 융합
- 24/7 가용성 요구
- 레거시 시스템과의 호환성
- 물리적 안전과 직결

### IIoT 보안 대책

- IT/OT 네트워크 분리
- 산업용 방화벽/IDS 적용
- 프로토콜 화이트리스트
- 변경 관리 절차 수립
- 물리적 접근 통제
