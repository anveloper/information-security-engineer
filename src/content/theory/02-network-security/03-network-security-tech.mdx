# 네트워크 보안 기술

네트워크를 보호하기 위한 다양한 보안 기술과 솔루션에 대해 학습합니다.

---

## 방화벽 (Firewall)

### 방화벽 개요

| 항목 | 설명 |
|------|------|
| 정의 | 네트워크 간 트래픽을 필터링하는 보안 장치 |
| 위치 | 네트워크 경계 (내부-외부, DMZ) |
| 기능 | 접근 제어, 로깅, NAT |

### 방화벽 유형

#### 패킷 필터링 방화벽 (Packet Filtering)

| 항목 | 설명 |
|------|------|
| 계층 | 네트워크/전송 계층 (L3/L4) |
| 필터 기준 | 출발지/목적지 IP, 포트, 프로토콜 |
| 장점 | 빠른 처리 속도, 간단한 구성 |
| 단점 | 애플리케이션 내용 검사 불가, 상태 추적 없음 |

**패킷 필터링 규칙 예시**

| 규칙 | 출발지 | 목적지 | 포트 | 프로토콜 | 동작 |
|------|--------|--------|------|----------|------|
| 1 | Any | 웹서버 | 80,443 | TCP | 허용 |
| 2 | 내부망 | Any | Any | Any | 허용 |
| 3 | Any | Any | Any | Any | 차단 |

#### 상태 기반 검사 방화벽 (Stateful Inspection)

| 항목 | 설명 |
|------|------|
| 계층 | 네트워크/전송 계층 (L3/L4) |
| 특징 | 연결 상태 테이블 유지 |
| 장점 | 연결 추적, 향상된 보안 |
| 상태 테이블 | 출발지, 목적지, 포트, 시퀀스 번호, 상태 |

**상태 테이블 예시**

| 출발지 IP | 출발지 포트 | 목적지 IP | 목적지 포트 | 상태 |
|----------|------------|----------|------------|------|
| 192.168.1.10 | 52341 | 8.8.8.8 | 443 | ESTABLISHED |
| 192.168.1.20 | 49152 | 1.1.1.1 | 80 | SYN_SENT |

#### 애플리케이션 게이트웨이 (Application Gateway / Proxy)

| 항목 | 설명 |
|------|------|
| 계층 | 응용 계층 (L7) |
| 동작 | 클라이언트-프록시-서버 중계 |
| 장점 | 깊은 패킷 검사, 콘텐츠 필터링, 캐싱 |
| 단점 | 처리 속도 느림, 프로토콜별 프록시 필요 |

#### 회선 레벨 게이트웨이 (Circuit-Level Gateway)

| 항목 | 설명 |
|------|------|
| 계층 | 세션 계층 (L5) |
| 동작 | TCP 연결 설정 검증 |
| 특징 | 연결 수립 후 패킷 내용 미검사 |
| 예시 | SOCKS 프록시 |

#### 차세대 방화벽 (NGFW)

| 항목 | 설명 |
|------|------|
| 통합 기능 | 방화벽 + IPS + 애플리케이션 인식 |
| 애플리케이션 제어 | 애플리케이션별 정책 적용 |
| 사용자 인식 | 사용자/그룹별 정책 |
| 위협 인텔리전스 | 실시간 위협 정보 활용 |
| SSL 검사 | 암호화 트래픽 복호화 검사 |

### 방화벽 아키텍처

#### 스크리닝 라우터 (Screening Router)

| 항목 | 설명 |
|------|------|
| 구성 | 라우터에 ACL 설정 |
| 장점 | 간단, 저비용 |
| 단점 | 제한된 필터링, 로깅 부족 |

#### 베스천 호스트 (Bastion Host)

| 항목 | 설명 |
|------|------|
| 정의 | 외부에 노출된 강화된 서버 |
| 특징 | 최소 서비스, 강력한 인증, 상세 로깅 |
| 용도 | 프록시, 점프 서버 |

#### 듀얼 홈드 호스트 (Dual-Homed Host)

| 항목 | 설명 |
|------|------|
| 구성 | 두 개의 네트워크 인터페이스 |
| 동작 | 내부-외부 네트워크 분리, 라우팅 비활성화 |
| 기능 | 프록시로 동작 |

#### 스크린드 호스트 (Screened Host)

| 항목 | 설명 |
|------|------|
| 구성 | 스크리닝 라우터 + 베스천 호스트 |
| 동작 | 라우터가 베스천으로만 트래픽 허용 |

#### 스크린드 서브넷 (Screened Subnet / DMZ)

| 항목 | 설명 |
|------|------|
| 구성 | 외부 방화벽 + DMZ + 내부 방화벽 |
| DMZ | 공개 서버 배치 영역 |
| 장점 | 다층 방어, 내부망 보호 |

```
인터넷 -- [외부 방화벽] -- DMZ -- [내부 방화벽] -- 내부망
                          |
                     웹서버, 메일서버
```

### 방화벽 정책

#### 정책 원칙

| 원칙 | 설명 |
|------|------|
| 기본 거부 (Default Deny) | 명시적으로 허용되지 않은 모든 트래픽 차단 |
| 최소 권한 | 필요한 최소한의 접근만 허용 |
| 심층 방어 | 여러 계층의 방화벽 구성 |

#### 규칙 순서

| 순서 | 설명 |
|------|------|
| 상단 | 자주 매칭되는 규칙 (성능) |
| 상단 | 명시적 차단 규칙 (보안) |
| 하단 | 기본 거부 규칙 |

### iptables

#### iptables 구조

| 테이블 | 용도 |
|--------|------|
| filter | 패킷 필터링 (기본) |
| nat | 주소 변환 |
| mangle | 패킷 수정 |
| raw | 연결 추적 예외 |

| 체인 (filter) | 설명 |
|---------------|------|
| INPUT | 로컬 시스템으로 들어오는 패킷 |
| OUTPUT | 로컬 시스템에서 나가는 패킷 |
| FORWARD | 시스템을 경유하는 패킷 |

#### iptables 명령어

```bash
# 규칙 확인
iptables -L -n -v

# 규칙 추가
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# 규칙 삽입
iptables -I INPUT 1 -p tcp --dport 80 -j ACCEPT

# 규칙 삭제
iptables -D INPUT -p tcp --dport 22 -j ACCEPT

# 체인 비우기
iptables -F INPUT

# 기본 정책 설정
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT
```

#### iptables 규칙 예시

```bash
# 루프백 허용
iptables -A INPUT -i lo -j ACCEPT

# 확립된 연결 허용
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# SSH 허용 (특정 IP)
iptables -A INPUT -s 192.168.1.0/24 -p tcp --dport 22 -j ACCEPT

# HTTP/HTTPS 허용
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# ICMP 허용 (제한)
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s -j ACCEPT

# 나머지 차단
iptables -A INPUT -j DROP
```

### nftables

iptables의 후속 도구로, 더 간결하고 효율적입니다.

```bash
# 테이블 생성
nft add table inet filter

# 체인 생성
nft add chain inet filter input { type filter hook input priority 0 \; policy drop \; }

# 규칙 추가
nft add rule inet filter input tcp dport 22 accept
nft add rule inet filter input tcp dport { 80, 443 } accept
```

## VPN (Virtual Private Network)

### VPN 개요

| 항목 | 설명 |
|------|------|
| 정의 | 공용 네트워크를 통한 암호화된 가상 사설 네트워크 |
| 목적 | 기밀성, 무결성, 인증 |
| 용도 | 원격 접속, 사이트 간 연결 |

### VPN 유형

| 유형 | 설명 |
|------|------|
| Remote Access VPN | 개별 사용자가 기업 네트워크에 접속 |
| Site-to-Site VPN | 지사 간 네트워크 연결 |
| Client-to-Site VPN | 클라이언트에서 VPN 게이트웨이로 연결 |
| SSL VPN | 웹 브라우저 기반 VPN |

### VPN 프로토콜

#### PPTP (Point-to-Point Tunneling Protocol)

| 항목 | 설명 |
|------|------|
| 포트 | TCP 1723, GRE (Protocol 47) |
| 암호화 | MPPE (RC4) |
| 인증 | MS-CHAPv2 |
| 보안 | 취약 (사용 비권장) |

#### L2TP/IPsec

| 항목 | 설명 |
|------|------|
| 계층 | L2 터널링 + IPsec 암호화 |
| 포트 | UDP 500, 4500, ESP (Protocol 50) |
| 암호화 | IPsec (AES) |
| 특징 | PPTP보다 안전, NAT 환경 이슈 |

#### IPsec

| 항목 | 설명 |
|------|------|
| 계층 | 네트워크 계층 (L3) |
| 모드 | 전송 모드, 터널 모드 |
| 프로토콜 | AH, ESP |
| 키 교환 | IKE (ISAKMP) |

**IPsec 모드**

| 모드 | 설명 | 용도 |
|------|------|------|
| 전송 모드 | IP 페이로드만 암호화 | 호스트 간 통신 |
| 터널 모드 | 전체 IP 패킷 암호화 + 새 IP 헤더 | VPN 게이트웨이 |

**IPsec 프로토콜**

| 프로토콜 | 기능 | Protocol 번호 |
|---------|------|--------------|
| AH (Authentication Header) | 인증, 무결성 (암호화 없음) | 51 |
| ESP (Encapsulating Security Payload) | 인증, 무결성, 암호화 | 50 |

**IKE (Internet Key Exchange)**

| 단계 | 설명 |
|------|------|
| Phase 1 | ISAKMP SA 설정 (Main/Aggressive 모드) |
| Phase 2 | IPsec SA 설정 (Quick 모드) |

| 모드 | 설명 |
|------|------|
| Main Mode | 6개 메시지, ID 보호 |
| Aggressive Mode | 3개 메시지, 빠름, ID 노출 |

#### SSL/TLS VPN

| 항목 | 설명 |
|------|------|
| 계층 | 응용/전송 계층 |
| 포트 | TCP 443 |
| 장점 | NAT 친화적, 방화벽 통과 용이 |
| 유형 | 클라이언트리스 (웹), 전체 터널 |

#### OpenVPN

| 항목 | 설명 |
|------|------|
| 프로토콜 | SSL/TLS 기반 |
| 포트 | UDP 1194 (기본), TCP 443 가능 |
| 암호화 | AES, Blowfish 등 |
| 인증 | 인증서, 사용자/패스워드 |
| 장점 | 오픈소스, 유연한 설정 |

#### WireGuard

| 항목 | 설명 |
|------|------|
| 특징 | 간단한 코드, 높은 성능 |
| 암호화 | ChaCha20, Curve25519 |
| 포트 | UDP |
| 장점 | 빠른 연결, 낮은 지연 |

### VPN 터널링

```
원본 패킷:
[IP Header | TCP/UDP | Data]

IPsec 터널 모드:
[New IP Header | ESP Header | IP Header | TCP/UDP | Data | ESP Trailer]

SSL VPN:
[IP Header | TCP | TLS | HTTP/Data]
```

### Split Tunneling

| 유형 | 설명 |
|------|------|
| Full Tunnel | 모든 트래픽이 VPN 경유 |
| Split Tunnel | 특정 트래픽만 VPN 경유, 나머지는 직접 |

| 항목 | Full Tunnel | Split Tunnel |
|------|-------------|--------------|
| 보안 | 높음 | 낮음 (로컬 위협 노출) |
| 성능 | VPN 부하 증가 | VPN 부하 감소 |
| 가시성 | 전체 트래픽 모니터링 | 일부만 모니터링 |

## IDS/IPS

### IDS (Intrusion Detection System)

| 항목 | 설명 |
|------|------|
| 정의 | 침입 탐지 시스템 |
| 동작 | 탐지 및 경고 (차단 안 함) |
| 배치 | 네트워크 탭/스팬 포트 (인라인 아님) |

### IPS (Intrusion Prevention System)

| 항목 | 설명 |
|------|------|
| 정의 | 침입 방지 시스템 |
| 동작 | 탐지 및 차단 |
| 배치 | 인라인 (트래픽 경로에 위치) |

### IDS vs IPS

| 항목 | IDS | IPS |
|------|-----|-----|
| 동작 | 탐지, 경고 | 탐지, 차단 |
| 배치 | 패시브 (미러링) | 인라인 |
| 지연 | 없음 | 있음 (처리 시간) |
| 장애 영향 | 없음 | 네트워크 중단 가능 |

### 탐지 유형

| 유형 | 설명 |
|------|------|
| NIDS/NIPS | 네트워크 기반 |
| HIDS/HIPS | 호스트 기반 |

#### NIDS (Network IDS)

| 항목 | 설명 |
|------|------|
| 위치 | 네트워크 세그먼트 |
| 모니터링 | 네트워크 트래픽 |
| 장점 | 네트워크 전체 가시성 |
| 단점 | 암호화 트래픽 분석 어려움 |

#### HIDS (Host IDS)

| 항목 | 설명 |
|------|------|
| 위치 | 개별 호스트 |
| 모니터링 | 시스템 콜, 파일, 로그 |
| 장점 | 암호화 트래픽 분석 가능 |
| 단점 | 호스트별 설치 필요 |

### 탐지 방식

#### 시그니처 기반 (Signature-based)

| 항목 | 설명 |
|------|------|
| 원리 | 알려진 공격 패턴 매칭 |
| 장점 | 정확한 탐지, 낮은 오탐 |
| 단점 | 제로데이 공격 탐지 불가, 시그니처 업데이트 필요 |

#### 이상 탐지 (Anomaly-based)

| 항목 | 설명 |
|------|------|
| 원리 | 정상 행위와의 편차 탐지 |
| 장점 | 알려지지 않은 공격 탐지 가능 |
| 단점 | 높은 오탐률, 학습 기간 필요 |

#### 정책 기반 (Policy-based)

| 항목 | 설명 |
|------|------|
| 원리 | 정의된 정책 위반 탐지 |
| 예시 | 허용되지 않은 프로토콜 사용 |

### Snort

오픈소스 네트워크 IDS/IPS입니다.

#### Snort 구성요소

| 구성요소 | 설명 |
|---------|------|
| Packet Decoder | 패킷 디코딩 |
| Preprocessors | 전처리 (정규화, 재조립) |
| Detection Engine | 규칙 매칭 |
| Output Plugins | 로깅, 경고 |

#### Snort 규칙 구조

```
[Action] [Protocol] [Source] [Port] -> [Dest] [Port] (Options)
```

**Snort 규칙 예시**

```
alert tcp any any -> any 80 (msg:"SQL Injection Attempt"; content:"' OR '1'='1"; nocase; sid:1000001; rev:1;)
```

| 부분 | 설명 |
|------|------|
| alert | 동작 (alert, log, pass, drop) |
| tcp | 프로토콜 |
| any any | 출발지 IP/포트 |
| `->` | 방향 |
| any 80 | 목적지 IP/포트 |
| msg | 경고 메시지 |
| content | 페이로드 패턴 |
| sid | 시그니처 ID |

#### Snort 규칙 옵션

| 옵션 | 설명 |
|------|------|
| msg | 경고 메시지 |
| content | 페이로드 패턴 |
| pcre | 정규표현식 |
| nocase | 대소문자 무시 |
| depth | 검색 깊이 |
| offset | 검색 시작 위치 |
| flow | 흐름 방향 (to_server, established) |
| sid | 시그니처 ID |
| rev | 리비전 |
| classtype | 분류 |
| priority | 우선순위 |

### Suricata

고성능 오픈소스 IDS/IPS/NSM입니다.

| 특징 | 설명 |
|------|------|
| 멀티스레드 | 고성능 처리 |
| 프로토콜 인식 | HTTP, TLS, DNS 등 |
| 파일 추출 | 네트워크에서 파일 추출 |
| Lua 스크립팅 | 유연한 탐지 규칙 |
| Snort 호환 | Snort 규칙 사용 가능 |

### 탐지 결과 분류

| 분류 | 설명 |
|------|------|
| True Positive (TP) | 실제 공격을 정확히 탐지 |
| True Negative (TN) | 정상을 정상으로 판단 |
| False Positive (FP) | 정상을 공격으로 오탐 |
| False Negative (FN) | 공격을 탐지하지 못함 (미탐) |

## 네트워크 접근 제어 (NAC)

### NAC 개요

| 항목 | 설명 |
|------|------|
| 정의 | 네트워크 접속 전 보안 상태 검증 |
| 목적 | 비인가/비준수 장치 접근 차단 |

### NAC 기능

| 기능 | 설명 |
|------|------|
| 인증 | 사용자/장치 신원 확인 |
| 상태 점검 | 패치, AV, 정책 준수 확인 |
| 격리 | 비준수 장치 격리 네트워크 |
| 치료 | 패치 설치, AV 업데이트 |
| 지속 모니터링 | 접속 후 상태 지속 확인 |

### NAC 구성요소

| 구성요소 | 설명 |
|---------|------|
| NAC 서버 | 정책 결정, 관리 |
| 에이전트 | 클라이언트 상태 수집 |
| 네트워크 장비 | 접근 제어 적용 (스위치, 무선 AP) |
| 치료 서버 | 패치, 업데이트 제공 |

### 802.1X

| 항목 | 설명 |
|------|------|
| 정의 | 포트 기반 네트워크 접근 제어 표준 |
| 구성요소 | Supplicant, Authenticator, Authentication Server |

**802.1X 구성요소**

| 구성요소 | 설명 |
|---------|------|
| Supplicant | 접속 요청 클라이언트 |
| Authenticator | 스위치/AP (접근 제어) |
| Authentication Server | RADIUS 서버 |

**802.1X 인증 과정**

```
Supplicant       Authenticator       RADIUS Server
    |                  |                   |
    |-- EAPOL-Start -->|                   |
    |<-- EAP-Request --|                   |
    |-- EAP-Response ->|                   |
    |                  |-- RADIUS Access-Request -->|
    |                  |<-- RADIUS Access-Challenge -|
    |<-- EAP-Request --|                   |
    |-- EAP-Response ->|                   |
    |                  |-- RADIUS Access-Request -->|
    |                  |<-- RADIUS Access-Accept ---|
    |<-- EAP-Success --|                   |
    |                  |                   |
    | 포트 열림 (인증 완료)                  |
```

### EAP (Extensible Authentication Protocol)

| EAP 유형 | 설명 |
|---------|------|
| EAP-MD5 | 단방향 인증 (취약) |
| EAP-TLS | 상호 인증서 기반 (가장 안전) |
| EAP-TTLS | 서버 인증서 + 터널 내 인증 |
| PEAP | 서버 인증서 + 터널 내 MSCHAPv2 |
| EAP-FAST | Cisco 개발, 빠른 재인증 |

### RADIUS

| 항목 | 설명 |
|------|------|
| 정의 | Remote Authentication Dial-In User Service |
| 포트 | UDP 1812 (인증), 1813 (계정) |
| 기능 | AAA (Authentication, Authorization, Accounting) |

## 용어 해설

| 용어 | 설명 |
|------|------|
| 방화벽 (Firewall) | 네트워크 간 트래픽을 정책에 따라 허용하거나 차단하는 보안 장치 |
| 패킷 필터링 (Packet Filtering) | IP, 포트, 프로토콜 등 패킷 헤더 정보만으로 트래픽을 필터링하는 방식. 상태 추적 없음 |
| 상태 기반 검사 (Stateful Inspection) | 연결 상태 테이블을 유지하여 세션 컨텍스트를 추적하며 필터링하는 방식 |
| NGFW (Next-Generation Firewall) | 전통적 방화벽에 IPS, 애플리케이션 인식, 사용자 인식 등을 통합한 차세대 방화벽 |
| 애플리케이션 게이트웨이 | L7 계층에서 동작하는 프록시 방식 방화벽. 깊은 패킷 검사 가능 |
| 회선 레벨 게이트웨이 | 세션 계층에서 TCP 연결만 검증하는 방화벽. SOCKS 프록시가 대표적 |
| DMZ (Demilitarized Zone) | 내부망과 외부망 사이의 중립 영역. 웹서버, 메일서버 등 공개 서버 배치 |
| 베스천 호스트 (Bastion Host) | 외부에 노출되어 방어 최전선 역할을 하는 강화된 서버. 최소 서비스, 강력한 인증 적용 |
| 듀얼 홈드 호스트 | 두 개의 네트워크 인터페이스를 가진 호스트. 내부-외부 네트워크를 분리하며 라우팅 비활성화 |
| 스크린드 서브넷 | 외부-DMZ-내부로 이어지는 다층 방화벽 구조. 가장 안전한 아키텍처 |
| ACL (Access Control List) | 라우터나 방화벽에서 트래픽 허용/차단 규칙을 정의한 목록 |
| iptables | Linux 커널 방화벽 관리 도구. filter, nat, mangle, raw 테이블로 구성 |
| nftables | iptables를 대체하는 차세대 Linux 방화벽 프레임워크 |
| VPN (Virtual Private Network) | 공용 네트워크를 통해 암호화된 가상 사설 네트워크를 구성하는 기술 |
| IPsec (Internet Protocol Security) | IP 계층에서 인증과 암호화를 제공하는 프로토콜 스위트 |
| AH (Authentication Header) | IPsec에서 인증과 무결성을 제공하는 프로토콜. 암호화 기능 없음 (Protocol 51) |
| ESP (Encapsulating Security Payload) | IPsec에서 인증, 무결성, 암호화를 모두 제공하는 프로토콜 (Protocol 50) |
| IKE (Internet Key Exchange) | IPsec에서 보안 연관(SA)을 협상하고 키를 교환하는 프로토콜 |
| SA (Security Association) | IPsec 통신을 위한 보안 매개변수 집합. SPI로 식별 |
| 전송 모드 (Transport Mode) | IPsec에서 IP 페이로드만 암호화하는 모드. 호스트 간 통신에 사용 |
| 터널 모드 (Tunnel Mode) | IPsec에서 전체 IP 패킷을 암호화하고 새 헤더를 추가하는 모드. VPN 게이트웨이 간 통신에 사용 |
| SSL VPN | SSL/TLS 기반의 VPN. 웹 브라우저로 접속 가능하며 NAT 환경에서도 잘 동작 |
| Split Tunneling | VPN 연결 시 일부 트래픽만 VPN을 경유하고 나머지는 직접 통신하는 방식 |
| IDS (Intrusion Detection System) | 네트워크 침입을 탐지하여 경고하는 시스템. 차단 기능 없음 |
| IPS (Intrusion Prevention System) | 네트워크 침입을 탐지하고 차단하는 시스템. 인라인 배치 |
| NIDS/NIPS | 네트워크 기반 침입 탐지/방지 시스템. 네트워크 트래픽 모니터링 |
| HIDS/HIPS | 호스트 기반 침입 탐지/방지 시스템. 시스템 콜, 파일, 로그 모니터링 |
| 시그니처 기반 탐지 | 알려진 공격 패턴과 매칭하여 탐지하는 방식. 제로데이 탐지 불가 |
| 이상 탐지 (Anomaly Detection) | 정상 행위에서 벗어난 편차를 탐지하는 방식. 알려지지 않은 공격 탐지 가능하나 오탐 높음 |
| True/False Positive/Negative | 탐지 결과 분류. TP(정탐), FP(오탐), TN(정상), FN(미탐) |
| Snort | 오픈소스 네트워크 IDS/IPS. 규칙 기반 탐지 엔진 |
| Suricata | 고성능 오픈소스 IDS/IPS/NSM. 멀티스레드 지원, Snort 규칙 호환 |
| Forward Proxy | 클라이언트 측에서 외부 서버로의 요청을 중계하는 프록시 |
| Reverse Proxy | 서버 측에서 클라이언트 요청을 내부 서버로 분배하는 프록시. 로드밸런싱, WAF 기능 |
| Transparent Proxy | 클라이언트 설정 없이 네트워크 레벨에서 트래픽을 가로채는 프록시 |
| NAC (Network Access Control) | 네트워크 접속 전 장치의 보안 상태를 검증하고 정책을 적용하는 솔루션 |
| 802.1X | 포트 기반 네트워크 접근 제어 IEEE 표준. Supplicant-Authenticator-Authentication Server 구조 |
| RADIUS (Remote Authentication Dial-In User Service) | AAA(인증, 인가, 계정) 서비스를 제공하는 프로토콜 |
| EAP (Extensible Authentication Protocol) | 다양한 인증 방식을 지원하는 확장 가능한 인증 프레임워크 |
| EAP-TLS | EAP 중 가장 안전한 방식. 클라이언트와 서버 양쪽 인증서 사용 |
| PEAP (Protected EAP) | 서버 인증서로 TLS 터널을 만들고 그 안에서 인증하는 EAP 방식 |
| 제로 트러스트 (Zero Trust) | "절대 신뢰하지 않고 항상 검증"하는 보안 모델. 내부 네트워크도 신뢰하지 않음 |
| 마이크로 세그멘테이션 | 워크로드 단위로 네트워크를 세분화하여 횡적 이동을 방지하는 기법 |
| WPA3-SAE | WPA3의 개인 모드에서 사용하는 인증 방식. Simultaneous Authentication of Equals |
| PMF (Protected Management Frames) | 무선 관리 프레임을 보호하여 Deauthentication 공격 방지. WPA3에서 필수 |
| SPF (Sender Policy Framework) | 이메일 발신 서버 IP를 DNS에 등록하여 발신자를 검증하는 방식 |
| DKIM (DomainKeys Identified Mail) | 이메일에 디지털 서명을 추가하여 발신자와 무결성을 검증하는 방식 |
| DMARC | SPF와 DKIM 결과를 바탕으로 이메일 처리 정책을 정의하는 표준 |
| BEC (Business Email Compromise) | 기업 임원 등을 사칭하여 금전이나 정보를 탈취하는 이메일 사기 |
