# 네트워크 기반 공격

네트워크 환경에서 발생하는 다양한 보안 위협과 공격 기법에 대해 학습합니다.

---

## 스니핑 (Sniffing)

### 스니핑 개요

| 항목 | 설명 |
|------|------|
| 정의 | 네트워크 트래픽을 도청하여 정보 수집 |
| 대상 | 패스워드, 세션 정보, 개인정보 등 |
| 환경 | 허브 네트워크, 스위치 네트워크 (공격 필요) |

### 스니핑 유형

| 유형 | 설명 |
|------|------|
| Passive Sniffing | 허브 환경에서 단순 도청 |
| Active Sniffing | 스위치 환경에서 공격을 통한 도청 |

### 스위치 환경 스니핑 기법

#### ARP Spoofing / ARP Poisoning

| 항목 | 설명 |
|------|------|
| 원리 | 위조된 ARP Reply로 ARP 캐시 변조 |
| 목적 | 중간자 위치 확보 (MITM) |
| 결과 | 트래픽이 공격자를 경유 |

**ARP Spoofing 동작**

```
정상 상태:
희생자 ARP 캐시: 게이트웨이 IP -> 게이트웨이 MAC

공격 후:
희생자 ARP 캐시: 게이트웨이 IP -> 공격자 MAC
```

**ARP Spoofing 대응**

| 대응 | 설명 |
|------|------|
| 정적 ARP 설정 | 중요 장비의 ARP 고정 |
| ARP 감시 도구 | arpwatch, XArp |
| Dynamic ARP Inspection | 스위치에서 ARP 검증 |
| 802.1X | 포트 기반 인증 |

#### MAC Flooding

| 항목 | 설명 |
|------|------|
| 원리 | 다량의 위조 MAC 주소로 MAC 테이블 오버플로우 |
| 결과 | 스위치가 허브처럼 동작 (Fail-Open) |
| 도구 | macof (dsniff) |

**MAC Flooding 대응**

| 대응 | 설명 |
|------|------|
| Port Security | 포트당 MAC 주소 수 제한 |
| 802.1X | 포트 기반 인증 |

#### Switch Jamming

스위치의 MAC 테이블을 가득 채워 스위치 기능을 마비시키는 공격입니다.

#### ICMP Redirect

| 항목 | 설명 |
|------|------|
| 원리 | 위조된 ICMP Redirect 메시지로 라우팅 변경 |
| 결과 | 트래픽이 공격자를 경유 |
| 대응 | ICMP Redirect 비활성화 |

```bash
# Linux ICMP Redirect 비활성화
sysctl -w net.ipv4.conf.all.accept_redirects=0
sysctl -w net.ipv4.conf.all.send_redirects=0
```

#### SPAN/Port Mirroring 악용

| 항목 | 설명 |
|------|------|
| SPAN | Switch Port Analyzer |
| 정상 용도 | 네트워크 모니터링, IDS 연결 |
| 악용 | 미러링 포트 접근 시 전체 트래픽 도청 |

### 스니핑 도구

| 도구 | 설명 |
|------|------|
| Wireshark | GUI 기반 패킷 분석기 |
| tcpdump | CLI 기반 패킷 캡처 |
| dsniff | 패스워드 스니핑 도구 |
| Ettercap | MITM 공격 도구 |
| Cain & Abel | Windows 스니핑/크래킹 도구 |

### 스니핑 대응

| 대응 | 설명 |
|------|------|
| 암호화 통신 | SSH, HTTPS, VPN 사용 |
| 스위치 보안 | Port Security, DAI |
| 네트워크 분리 | VLAN 분리 |
| 모니터링 | ARP 변화 감시 |

## 스푸핑 (Spoofing)

### 스푸핑 개요

| 항목 | 설명 |
|------|------|
| 정의 | 신원을 위조하여 다른 시스템으로 가장 |
| 목적 | 인증 우회, 공격 출처 은닉, 서비스 방해 |

### 스푸핑 유형

#### IP Spoofing

| 항목 | 설명 |
|------|------|
| 원리 | 출발지 IP 주소 위조 |
| 용도 | DoS 공격, 신원 은닉 |
| 한계 | 응답 수신 불가 (Blind Spoofing) |

**IP Spoofing 대응**

| 대응 | 설명 |
|------|------|
| Ingress Filtering | 내부 IP를 출발지로 한 외부 패킷 차단 |
| Egress Filtering | 외부 IP를 출발지로 한 내부 패킷 차단 |
| uRPF | Unicast Reverse Path Forwarding |

#### ARP Spoofing

앞서 스니핑에서 설명한 ARP 캐시 변조 공격입니다.

#### DNS Spoofing

| 항목 | 설명 |
|------|------|
| 원리 | 위조된 DNS 응답 전송 |
| 결과 | 사용자가 악성 사이트로 접속 |
| 유형 | DNS Cache Poisoning, 로컬 DNS 변조 |

**DNS Cache Poisoning**

```
정상 DNS 응답보다 먼저 위조 응답 전송
  -> DNS 서버 캐시에 위조 레코드 저장
  -> 이후 모든 클라이언트가 위조된 IP로 접속
```

**DNS Spoofing 대응**

| 대응 | 설명 |
|------|------|
| DNSSEC | DNS 응답 서명 검증 |
| DNS over HTTPS (DoH) | HTTPS로 DNS 암호화 |
| DNS over TLS (DoT) | TLS로 DNS 암호화 |
| 랜덤 소스 포트 | 예측 공격 방지 |
| 랜덤 트랜잭션 ID | 위조 응답 매칭 방지 |

#### DHCP Spoofing

| 항목 | 설명 |
|------|------|
| 원리 | 악성 DHCP 서버 운영 |
| 결과 | 잘못된 게이트웨이, DNS 설정 |
| 목적 | MITM 공격, DNS 스푸핑 |

**DHCP Spoofing 대응**

| 대응 | 설명 |
|------|------|
| DHCP Snooping | 신뢰할 수 있는 DHCP 서버만 허용 |
| 정적 IP 설정 | 중요 장비에 고정 IP |

#### Email Spoofing

| 항목 | 설명 |
|------|------|
| 원리 | 발신자 주소 위조 |
| 용도 | 피싱, 스팸, 사기 |

**Email Spoofing 대응**

| 대응 | 설명 |
|------|------|
| SPF | 발신 서버 IP 검증 |
| DKIM | 이메일 서명 검증 |
| DMARC | SPF + DKIM 정책 |

## 세션 하이재킹 (Session Hijacking)

### 세션 하이재킹 개요

| 항목 | 설명 |
|------|------|
| 정의 | 인증된 세션을 가로채어 권한 탈취 |
| 대상 | TCP 연결, 웹 세션, 인증 토큰 |

### TCP 세션 하이재킹

| 단계 | 설명 |
|------|------|
| 1 | 스니핑으로 TCP 시퀀스 번호 획득 |
| 2 | 원래 클라이언트 연결 끊기 (RST 또는 DoS) |
| 3 | 시퀀스 번호 예측하여 세션 가로채기 |

**TCP 세션 하이재킹 대응**

| 대응 | 설명 |
|------|------|
| 암호화 | SSH, TLS 사용 |
| 랜덤 ISN | 시퀀스 번호 예측 방지 |
| 상호 인증 | 양방향 인증 |

### 웹 세션 하이재킹

#### 세션 ID 탈취 방법

| 방법 | 설명 |
|------|------|
| 스니핑 | 네트워크에서 세션 쿠키 도청 |
| XSS | 스크립트로 쿠키 탈취 |
| Session Fixation | 공격자가 설정한 세션 ID 사용 유도 |
| 예측 | 취약한 세션 ID 생성 알고리즘 악용 |

#### Session Fixation

```
1. 공격자가 사이트에서 세션 ID 획득
2. 희생자에게 해당 세션 ID로 접속 유도
3. 희생자가 로그인
4. 공격자가 같은 세션 ID로 접속 -> 인증된 상태
```

**Session Fixation 대응**

| 대응 | 설명 |
|------|------|
| 로그인 시 세션 ID 재생성 | 인증 후 새 세션 발급 |
| 세션 ID를 URL에 포함 금지 | 쿠키만 사용 |

### 세션 하이재킹 대응

| 대응 | 설명 |
|------|------|
| HTTPS 사용 | 세션 정보 암호화 |
| HttpOnly 쿠키 | JavaScript 접근 차단 |
| Secure 쿠키 | HTTPS에서만 전송 |
| SameSite 쿠키 | CSRF 방지 |
| 세션 타임아웃 | 유휴 세션 만료 |
| IP 바인딩 | 세션을 IP에 연결 |

## 서비스 거부 공격 (DoS/DDoS)

### DoS 개요

| 항목 | 설명 |
|------|------|
| 정의 | 시스템 자원을 고갈시켜 서비스 불가 상태 유발 |
| DoS | 단일 공격자 |
| DDoS | 다수의 공격자 (봇넷) |

### DoS 공격 유형

#### 대역폭 소진 공격

| 공격 | 설명 |
|------|------|
| UDP Flood | 대량의 UDP 패킷 전송 |
| ICMP Flood | 대량의 ICMP Echo Request 전송 |
| Amplification | 증폭 공격 (DNS, NTP, SSDP) |

#### 프로토콜 취약점 공격

| 공격 | 설명 |
|------|------|
| SYN Flood | 대량의 SYN 패킷으로 연결 대기 큐 고갈 |
| ACK Flood | 대량의 ACK 패킷 전송 |
| TCP Connection Flood | 다수의 TCP 연결 수립 |

#### 애플리케이션 공격

| 공격 | 설명 |
|------|------|
| HTTP Flood | 대량의 HTTP 요청 |
| Slowloris | 느린 HTTP 요청으로 연결 점유 |
| RUDY | 느린 POST 데이터 전송 |

### 주요 DoS/DDoS 공격

#### SYN Flood

| 항목 | 설명 |
|------|------|
| 원리 | SYN 패킷만 보내고 ACK 안 함 |
| 결과 | 서버의 연결 대기 큐(Backlog) 고갈 |
| 특징 | 출발지 IP 위조 (응답 불필요) |

```
공격자 -> SYN (src: 위조 IP) -> 서버
서버 -> SYN+ACK -> 위조 IP (응답 없음)
서버: SYN_RECEIVED 상태로 대기 (자원 점유)
```

**SYN Flood 대응**

| 대응 | 설명 |
|------|------|
| SYN Cookies | 상태 정보 없이 연결 처리 |
| Backlog 큐 증가 | 대기 큐 크기 확대 |
| SYN 타임아웃 감소 | 대기 시간 단축 |
| 방화벽/IPS | SYN Flood 탐지 및 차단 |

#### Smurf Attack

| 항목 | 설명 |
|------|------|
| 원리 | ICMP Echo Request를 브로드캐스트 주소로 전송 (출발지: 희생자 IP) |
| 결과 | 네트워크의 모든 호스트가 희생자에게 Echo Reply |
| 증폭 | 1:N 증폭 효과 |

**Smurf Attack 대응**

| 대응 | 설명 |
|------|------|
| Directed Broadcast 비활성화 | 라우터에서 차단 |
| ICMP 응답 비활성화 | 호스트 설정 |
| Ingress Filtering | 스푸핑 차단 |

#### Fraggle Attack

| 항목 | 설명 |
|------|------|
| 원리 | UDP Echo (7번 포트)를 브로드캐스트로 전송 |
| 특징 | Smurf의 UDP 버전 |
| 대응 | UDP Echo 비활성화 |

#### Land Attack

| 항목 | 설명 |
|------|------|
| 원리 | 출발지와 목적지 IP/포트를 동일하게 설정 |
| 결과 | 시스템이 자기 자신에게 응답 시도 |
| 대응 | 최신 OS 패치, 방화벽 필터링 |

#### Teardrop Attack

| 항목 | 설명 |
|------|------|
| 원리 | 조각난 IP 패킷의 오프셋을 겹치게 설정 |
| 결과 | 재조합 시 시스템 충돌 |
| 대응 | OS 패치 |

#### Ping of Death

| 항목 | 설명 |
|------|------|
| 원리 | 65,535바이트 초과 ICMP 패킷 전송 |
| 결과 | 버퍼 오버플로우로 시스템 충돌 |
| 대응 | OS 패치 |

### 증폭 공격 (Amplification Attack)

#### DNS Amplification

| 항목 | 설명 |
|------|------|
| 원리 | DNS 쿼리 (작음) -> DNS 응답 (큼) |
| 증폭 비율 | 최대 50배 이상 |
| 방법 | 오픈 리졸버에 ANY 쿼리 (출발지: 희생자 IP) |

#### NTP Amplification

| 항목 | 설명 |
|------|------|
| 원리 | monlist 요청으로 대량 응답 |
| 증폭 비율 | 최대 500배 이상 |
| 대응 | monlist 비활성화, NTP 버전 업데이트 |

#### SSDP Amplification

| 항목 | 설명 |
|------|------|
| 원리 | SSDP M-SEARCH 요청 |
| 증폭 비율 | 약 30배 |
| 대응 | 외부 SSDP 차단 |

#### Memcached Amplification

| 항목 | 설명 |
|------|------|
| 원리 | Memcached 프로토콜 악용 |
| 증폭 비율 | 최대 50,000배 이상 |
| 대응 | 외부 접근 차단, 인증 설정 |

### DDoS 공격 인프라

#### 봇넷 (Botnet)

| 구성요소 | 설명 |
|---------|------|
| Bot | 악성코드에 감염된 좀비 PC |
| C&C Server | 명령 및 제어 서버 |
| Bot Master | 공격자 |

**봇넷 C&C 방식**

| 방식 | 설명 |
|------|------|
| IRC | IRC 채널을 통한 명령 전달 |
| HTTP | 웹 서버 기반 명령 전달 |
| P2P | 탈중앙화된 명령 전달 |
| DNS | DNS를 통한 명령 전달 |

### DDoS 대응

| 계층 | 대응 |
|------|------|
| 네트워크 | 대역폭 확보, ISP 협조, Blackhole Routing |
| 인프라 | CDN, 로드밸런서, Anycast |
| 애플리케이션 | WAF, Rate Limiting, CAPTCHA |
| 서비스 | DDoS 방어 서비스 (Cloudflare, AWS Shield) |

## 중간자 공격 (MITM)

### MITM 개요

| 항목 | 설명 |
|------|------|
| 정의 | 통신 중간에서 메시지 도청/변조 |
| 위치 | 클라이언트와 서버 사이 |
| 목적 | 정보 탈취, 데이터 변조, 세션 하이재킹 |

### MITM 공격 기법

| 기법 | 설명 |
|------|------|
| ARP Spoofing | MAC 주소 위조로 중간자 위치 확보 |
| DNS Spoofing | DNS 응답 위조로 트래픽 우회 |
| SSL Stripping | HTTPS를 HTTP로 다운그레이드 |
| Evil Twin | 악성 Wi-Fi AP 운영 |
| BGP Hijacking | 라우팅 경로 조작 |

### SSL Stripping

| 단계 | 설명 |
|------|------|
| 1 | 클라이언트가 HTTP로 접속 |
| 2 | 서버가 HTTPS 리다이렉트 |
| 3 | 공격자가 리다이렉트 가로채기 |
| 4 | 공격자-서버: HTTPS, 공격자-클라이언트: HTTP |

```
클라이언트 <-- HTTP --> 공격자 <-- HTTPS --> 서버
```

**SSL Stripping 대응**

| 대응 | 설명 |
|------|------|
| HSTS | HTTP Strict Transport Security |
| HSTS Preload | 브라우저에 HTTPS 강제 목록 등록 |

### Evil Twin Attack

| 항목 | 설명 |
|------|------|
| 원리 | 합법적 AP와 동일한 SSID로 악성 AP 운영 |
| 결과 | 사용자가 악성 AP에 접속 |
| 활동 | 트래픽 도청, DNS 스푸핑, 피싱 |

**Evil Twin 대응**

| 대응 | 설명 |
|------|------|
| 인증서 확인 | HTTPS 인증서 검증 |
| VPN 사용 | 공공 Wi-Fi에서 VPN |
| 802.1X | 기업 환경 무선 인증 |

## 스캐닝 공격

### 포트 스캐닝

#### 스캔 유형

| 스캔 | 플래그 | 응답 (열림) | 응답 (닫힘) | 특징 |
|------|--------|------------|------------|------|
| TCP Connect | SYN | SYN+ACK | RST | 완전 연결, 로그 남음 |
| SYN (Half-Open) | SYN | SYN+ACK | RST | 스텔스, 빠름 |
| FIN | FIN | 응답 없음 | RST | 방화벽 우회 시도 |
| NULL | 없음 | 응답 없음 | RST | 방화벽 우회 시도 |
| XMAS | FIN+PSH+URG | 응답 없음 | RST | 방화벽 우회 시도 |
| ACK | ACK | RST | RST | 방화벽 규칙 탐지 |
| UDP | - | 응답 없음/응답 | ICMP Port Unreachable | 느림 |

## 용어 해설

| 용어 | 설명 |
|------|------|
| 스니핑 (Sniffing) | 네트워크 트래픽을 도청하여 정보를 수집하는 공격. 패시브(허브 환경)와 액티브(스위치 환경) 방식으로 구분 |
| ARP Spoofing/Poisoning | 위조된 ARP Reply를 보내 희생자의 ARP 캐시를 변조하는 공격. MITM 공격의 전 단계로 활용 |
| MAC Flooding | 다량의 위조 MAC 주소로 스위치의 MAC 테이블을 오버플로우시켜 스위치를 허브처럼 동작하게 만드는 공격 |
| DAI (Dynamic ARP Inspection) | 스위치에서 ARP 패킷을 검증하여 ARP Spoofing을 방지하는 기능 |
| SPAN (Switch Port Analyzer) | 스위치에서 특정 포트의 트래픽을 다른 포트로 미러링하는 기능. 네트워크 모니터링에 사용 |
| Promiscuous Mode | 네트워크 인터페이스가 자신에게 오지 않은 패킷도 모두 수신하는 모드. 스니핑에 필요 |
| IP Spoofing | 패킷의 출발지 IP 주소를 위조하는 공격. DoS 공격이나 신원 은닉에 사용 |
| DNS Cache Poisoning | DNS 서버의 캐시에 위조된 레코드를 삽입하여 사용자를 악성 사이트로 유도하는 공격 |
| Ingress/Egress Filtering | 네트워크 경계에서 위조된 IP 주소를 가진 패킷을 차단하는 필터링 기법 |
| uRPF (Unicast Reverse Path Forwarding) | 패킷의 출발지 IP가 해당 인터페이스로 라우팅 가능한지 검증하여 IP Spoofing을 방지 |
| Session Hijacking | 인증된 사용자의 세션을 가로채어 권한을 탈취하는 공격 |
| Session Fixation | 공격자가 미리 설정한 세션 ID를 희생자가 사용하도록 유도하는 공격 |
| DoS (Denial of Service) | 시스템 자원을 고갈시켜 정상적인 서비스를 방해하는 공격 |
| DDoS (Distributed DoS) | 다수의 공격자(봇넷)를 이용한 분산 서비스 거부 공격 |
| SYN Flood | TCP 연결 과정에서 SYN 패킷만 대량 전송하여 서버의 연결 대기 큐를 고갈시키는 공격 |
| SYN Cookies | SYN Flood 방어 기법. 상태 정보 없이 쿠키 값으로 연결을 검증 |
| Backlog Queue | TCP 연결 요청을 대기시키는 큐. SYN Flood 공격의 대상 |
| Smurf Attack | ICMP Echo Request를 브로드캐스트로 보내 희생자에게 대량의 Echo Reply가 집중되게 하는 공격 |
| Fraggle Attack | Smurf Attack의 UDP 버전. UDP Echo를 브로드캐스트로 전송 |
| Land Attack | 출발지와 목적지를 동일하게 설정하여 시스템이 자기 자신에게 응답하게 만드는 공격 |
| Teardrop Attack | IP 단편화 패킷의 오프셋을 조작하여 재조합 시 시스템 충돌을 유발하는 공격 |
| Ping of Death | 65,535바이트를 초과하는 ICMP 패킷을 전송하여 버퍼 오버플로우를 일으키는 공격 |
| Amplification Attack | DNS, NTP 등의 프로토콜 특성을 악용하여 작은 요청으로 큰 응답을 유발하는 증폭 공격 |
| Open Resolver | 누구나 쿼리할 수 있는 DNS 서버. DNS Amplification 공격에 악용됨 |
| monlist | NTP 서버에서 최근 접속 클라이언트 목록을 반환하는 명령. NTP Amplification에 악용 |
| Botnet | 악성코드에 감염된 다수의 좀비 PC로 구성된 네트워크. DDoS 공격에 활용 |
| C&C (Command and Control) | 봇넷을 제어하기 위한 명령 및 제어 서버 |
| MITM (Man-in-the-Middle) | 통신하는 두 당사자 사이에서 메시지를 도청하거나 변조하는 공격 |
| SSL Stripping | HTTPS 연결을 HTTP로 다운그레이드하여 평문 통신을 유도하는 MITM 공격 |
| HSTS Preload | 브라우저에 미리 HTTPS를 강제하는 도메인 목록을 등록하여 SSL Stripping 방지 |
| Evil Twin | 정상 AP와 동일한 SSID로 악성 AP를 운영하여 사용자를 속이는 공격 |
| 포트 스캐닝 (Port Scanning) | 대상 시스템의 열린 포트와 서비스를 탐색하는 정찰 활동 |
| SYN Scan (Half-Open) | SYN 패킷만 보내고 연결을 완료하지 않는 스텔스 스캔 기법 |
| XMAS Scan | FIN, PSH, URG 플래그를 모두 설정한 패킷으로 스캔하는 기법 |
| WEP Cracking | IV(Initialization Vector) 취약점을 이용하여 WEP 키를 추출하는 공격 |
| KRACK (Key Reinstallation Attack) | WPA2의 4-Way Handshake 취약점을 악용하는 공격 |
| Deauthentication Attack | 위조된 인증 해제 프레임을 전송하여 무선 클라이언트 연결을 끊는 공격 |
| PMKID Attack | 4-Way Handshake 캡처 없이 WPA/WPA2 PSK를 공격하는 기법 |
| DNS Tunneling | DNS 쿼리/응답에 데이터를 인코딩하여 방화벽을 우회하고 데이터를 유출하는 기법 |
| BGP Hijacking | 잘못된 BGP 경로를 광고하여 인터넷 트래픽을 가로채거나 차단하는 공격 |
| RPKI (Resource Public Key Infrastructure) | BGP 경로 광고의 진위를 검증하기 위한 PKI 기반 인프라 |
| Port Knocking | 미리 정의된 포트 시퀀스로 접속해야 실제 포트가 열리는 보안 기법 |
